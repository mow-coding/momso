> 작성일: 2026-06-08

## 0. 한 줄 요약

자문가 피드백의 핵심은 **momso가 왜 특정 데이터를 반드시 써야 하는지, 특히 왜 인바디 데이터가 필요한지의 필연성이 아직 약하다**는 점이다.

발표자료는 단순 슬라이드 나열보다 **전체 그림, 목차, 논리 지도, 근거자료 인덱스, 팀 서사**가 먼저 보이도록 재구성해야 한다. 또한 시장 레퍼런스만이 아니라 **AC/투자사 포트폴리오 기업과 특허 레퍼런스**까지 다시 확인해야 한다.

---

## 1. 제품/데이터 논리 피드백

### 1-1. 특정 데이터 사용의 필연성이 부족함

#### 피드백

현재 momso 설명에서는 **왜 이 서비스가 특정 형태의 데이터를 반드시 사용해야 하는지**가 충분히 설득되지 않는다.

예를 들어, 수업 녹음 음성데이터만 있다고 하면 서비스가 허술해 보일 수 있다. "수업을 녹음해서 AI로 정리한다" 정도로 보이면, 단순 전사/회의록 앱과의 차이가 약해진다.

#### 의미

지금의 발표 논리는 "수업 맥락 기록이 필요하다"까지는 말하고 있지만, 그 맥락 기록을 만들기 위해 어떤 데이터 조합이 필수인지 더 선명하게 보여줘야 한다.

특히 아래 질문에 답해야 한다.

- momso가 다루는 핵심 데이터는 정확히 무엇인가?
- 음성 데이터만으로 충분하지 않다면, 무엇이 더 필요한가?
- 강사 검수 데이터, 수련생 감각 메모, 인바디 수치, 수업 이력은 각각 어떤 역할을 하는가?
- 이 데이터들이 합쳐질 때만 가능한 결과물은 무엇인가?

#### 반영 액션

- 지원서/덱에 **데이터 필연성 섹션**을 추가한다.
- 단순히 "수업 녹음"이 아니라 다음 데이터 조합을 구조화한다.

| 데이터 | 역할 | 단독 사용 시 한계 | momso에서 필요한 이유 |
|---|---|---|---|
| 수업 음성/전사 | 수업 중 실제 언어와 교정 기록 | 단순 회의록처럼 보일 수 있음 | 지도자의 말과 피드백을 포착하는 출발점 |
| 강사 검수 데이터 | 공유 가능/내부/제외 판단 | 사람이 개입하지 않으면 민감정보와 오해 위험 | 요가 도메인에서 안전성과 신뢰를 만드는 핵심 |
| 수련생 감각 메모 | 본인이 느낀 변화 | 주관적이고 흩어질 수 있음 | 수치로 설명되지 않는 웰니스 변화를 기록 |
| 인바디 데이터 | 몸의 정량 참고값 | 수업 맥락 없이 보면 숫자만 남음 | 수업 전후 변화를 눈에 보이게 하는 정량 anchor |
| 수업 이력/회원별 아카이브 | 장기 변화 흐름 | 개별 수업만 보면 축적 효과가 약함 | 다음 수업 준비와 재방문 상담의 근거 |

---

### 1-2. "인바디 데이터가 왜 필요한가"가 약함

#### 피드백

InBodyLIKE에서 가장 중요한 질문은 **"왜 인바디 데이터가 필요한가?"**인데, 현재 자료에서는 이 부분이 약하다.

momso가 인바디와 협업하고 싶다는 말은 있으나, 인바디 데이터가 없으면 무엇이 안 되는지, 인바디 데이터가 들어오면 무엇이 강해지는지가 더 선명해야 한다.

#### 의미

지금의 표현은 "인바디는 첫 정량 데이터 파트너"라는 수준에 머물 수 있다. 심사자 입장에서는 "그럼 인바디 없이도 되는 서비스 아닌가?", "왜 InBodyLIKE에 적합한가?"라고 물을 수 있다.

따라서 인바디 데이터의 역할을 다음처럼 다시 정의해야 한다.

- 인바디 데이터는 momso의 전부가 아니다.
- 그러나 인바디 데이터는 **수업 맥락 기록이 실제 몸의 변화와 연결되는지 검증하는 정량 anchor**다.
- 인바디 데이터가 있으면 "좋은 수업이었다"에서 끝나는 것이 아니라, 수업 전후 몸 상태의 변화를 참고값으로 함께 볼 수 있다.
- 이때 momso는 인바디 데이터를 진단/처방으로 쓰는 것이 아니라, 수업 언어와 감각 변화의 맥락 안에 배치한다.

#### 반영 액션

- 발표자료에 "왜 인바디 데이터인가" 슬라이드를 별도로 만든다.
- 인바디 데이터의 필요성을 아래 논리로 재구성한다.

```text
문제: 수업의 변화는 느껴지지만 보이지 않는다.
인바디: 변화의 일부를 정량 참고값으로 보이게 한다.
momso: 그 정량 변화가 어떤 수업 언어, 피드백, 감각 변화와 함께 나타났는지 설명한다.
결과: 정량 데이터와 정성 수업 맥락이 함께 있는 웰니스 리포트가 된다.
```

#### 수정해야 할 질문

기존 질문:

> 인바디와 협업하려면 무엇을 먼저 요청해야 하나?

수정된 질문:

> momso의 서비스 가치에서 인바디 데이터가 빠지면 무엇이 약해지고, 들어오면 어떤 검증이 가능해지는가?

## 2. 발표자료 구성 피드백

### 2-1. 목차와 전체 그림이 필요함

#### 피드백

발표자료에 **목차**가 필요하다. 또한 전체 발표가 어떤 논리로 구성되어 있는지 한눈에 볼 수 있는 **전체 그림/논리 지도**가 필요하다.

#### 의미

현재 덱은 각 슬라이드 내용은 있으나, 심사자가 "이 발표가 어디로 가고 있는지"를 한눈에 잡기 어렵다. 특히 문제, 데이터, BM, 인바디 협업, 팀 서사가 각각 따로 보이면 설득력이 떨어진다.

#### 반영 액션

발표 초반에 아래 구조를 추가한다.

1. 목차 슬라이드
2. 전체 논리 지도 슬라이드
3. 핵심 근거자료 인덱스 슬라이드

예상 논리 지도:

```text
팀 배경
  -> 현장에서 발견한 문제
  -> 왜 수업 데이터가 필요한가
  -> 왜 인바디 데이터가 필요한가
  -> momso의 데이터 조합과 HITL
  -> PoC 설계
  -> BM/시장 확장
  -> 요청 사항
```

---

### 2-2. 슬라이드별 근거자료 하이퍼링크 방식 변경

#### 피드백

슬라이드마다 하이퍼링크로 근거자료를 붙이는 방식보다, **목차처럼 하나의 슬라이드에 근거자료를 모아 보여주는 방식**이 낫다.

#### 의미

슬라이드마다 링크가 붙으면 발표 흐름이 산만해질 수 있다. 심사자는 세부 링크보다 "이 팀이 어떤 근거 묶음을 갖고 있는지"를 먼저 보고 싶어할 수 있다.

#### 반영 액션

- 각 슬라이드 footer의 긴 출처 문구는 줄인다.
- 별도 슬라이드 또는 appendix에 **근거자료 인덱스**를 만든다.
- 근거자료 인덱스는 아래처럼 구성한다.

| 근거 묶음 | 설명 | 대표 자료 |
|---|---|---|
| 도메인 실행력 | 유동환/빅블루의 현장 경험 | 연희 요가 위크 공개 보고서 |
| 시장/BM | 가격, 시장, 매출 시나리오 검증 | BM 리서치 문서 |
| 경쟁 | CRM, 예약앱, 체형분석, 전사앱 비교 | 경쟁사 리서치 문서 |
| 인바디 연동 | API, 앱, 결과지 업로드, PoC 현실성 | 인바디 연동 리서치 |
| 개인정보/AI | 녹음, 동의, 검수, 건강정보 리스크 | 개인정보/AI 처리 근거 |
| 특허/AC 레퍼런스 | 새로 보강 필요 | 미수행 |

---

### 2-3. 팀 소개를 맨 앞에 배치하고 필모그래픽식으로 구성

#### 피드백

팀 소개는 맨 마지막보다 **맨 처음 쪽에 배치**하는 것이 좋다. 단순한 이력 나열이 아니라, **필모그래픽식으로 구성**하고 팀 배경이 아이템으로 자연스럽게 이어지도록 서사적으로 구성해야 한다.

#### 의미

momso는 팀의 현장성이 중요한 아이템이다. 유동환이 요가원을 운영하고, 연희 요가 위크를 운영했고, 김성균이 기술/데이터 구조를 만들었다는 흐름이 먼저 보여야 아이템의 출발점이 자연스럽다.

즉 "우리가 이런 팀입니다"가 아니라:

> 이런 현장을 겪은 팀이어서, 이 문제를 발견했고, 그래서 momso가 나왔다.

라고 보여줘야 한다.

#### 반영 액션

발표 초반에 팀 서사 슬라이드를 추가/재배치한다.

예상 구성:

```text
유동환
- 빅블루 요가 운영
- 연희 요가 위크 운영
- 수련생 피드백과 수업 맥락이 사라지는 문제를 현장에서 경험

김성균
- 기술/데이터 구조 설계
- momso 웹 프로토타입 구현
- 현장 기록을 데이터 구조와 서비스 흐름으로 전환

두 사람의 접점
- 현장의 암묵지와 기술적 기록 계층을 연결
- 그래서 momso는 단순 녹음앱이 아니라 수업 맥락 기록 서비스
```

## 3. 레퍼런스 체크 추가 과제

### 3-1. 시장 레퍼런스만으로 부족함

#### 피드백

레퍼런스 체크를 시장 규모나 경쟁사 관점에서만 하면 부족하다. 한 번 더 해야 한다.

#### 추가로 봐야 할 방향

1. AC 산하 또는 투자받고 있는 기업
2. 헬스케어/웰니스/AI 코칭/스튜디오 SaaS 관련 포트폴리오 기업
3. momso 기능 정의와 대조되는 특허

---

### 3-2. AC/투자 포트폴리오 레퍼런스

#### 의미

InBodyLIKE는 단순 시장성만 보는 프로그램이 아니라, 액셀러레이터/투자 관점에서 아이템의 차별성과 확장성을 볼 가능성이 크다.

따라서 다음을 확인해야 한다.

- 유사한 헬스케어 데이터 스타트업이 어떤 문제를 풀고 있는가
- AI 코칭/건강관리/운동 데이터 기업들은 어떤 데이터 조합을 쓰는가
- 투자받은 기업들은 자신의 핵심 데이터 필연성을 어떻게 설명하는가
- momso와 겹치는 기업이 있는가
- momso가 다르게 가져가야 할 포지션은 무엇인가

#### 반영 액션

새 리서치 과제로 분리한다.

| 조사 대상 | 볼 것 |
|---|---|
| 블루포인트 포트폴리오 | 헬스케어, 웰니스, AI, B2B SaaS 기업 중 유사 사례 |
| AC/VC 투자 기업 | 운동 기록, AI 코칭, 건강 데이터, 센터 SaaS 관련 기업 |
| InBodyLIKE 관련 스타트업 | 인바디 데이터 활용 방향과 겹치는지 |
| 국내외 웰니스 데이터 기업 | 데이터 필연성, PoC 구조, BM |

---

### 3-3. 특허 레퍼런스 체크

#### 피드백

momso의 기능 정의와 대조해서 **특허 쪽으로도 한 번 더 돌려야 한다**.

#### 의미

특허 조사는 단순히 "특허를 낼 수 있나"만 보기 위한 것이 아니다. 이미 어떤 회사들이 유사한 기능을 정의했는지, 어떤 데이터 조합이 기존에 주장되었는지, momso가 피해야 할 표현이나 차별화해야 할 지점을 확인하는 데 필요하다.

#### 조사 기준 초안

| momso 기능 | 특허/기술 검색 포인트 |
|---|---|
| 수업 음성 전사 | 운동/요가/피트니스 수업 음성 기록, 코칭 발화 분석 |
| 강사 검수 | AI 생성 리포트의 human review, instructor approval workflow |
| 개인 수련 리포트 | exercise session report, wellness progress report |
| 인바디/체성분 결합 | body composition data + exercise/coaching recommendation |
| 수련생 감각 메모 | subjective body sensation, self-reported wellness data |
| 다음 수업 환류 | adaptive coaching, longitudinal fitness instruction |
| 데이터 주권/BYOK | user-owned health data, personal data vault, consent-based sharing |

#### 반영 액션

- 별도 리서치 문서로 `특허/AC 레퍼런스 체크`를 만든다.
- 지원서에는 아직 조사 전임을 표시하고, 확인 후 차별점에 반영한다.

## 5. 현재 메모 기준으로 바뀌는 핵심 질문

자문 전에는 아래 질문이 중심이었다.

> 인바디와 어떻게 협업할 것인가?

자문 후에는 질문을 더 앞단으로 당겨야 한다.

> momso가 왜 이 데이터 조합을 반드시 필요로 하며, 그중 인바디 데이터는 어떤 검증을 가능하게 하는가?

이 질문에 답해야 InBodyLIKE 적합성이 선명해진다.

## 6. 후속 작업 제안

1. 이 메모를 바탕으로 PPT 구조를 재배치한다.
2. `데이터 필연성`과 `왜 인바디 데이터인가`를 별도 섹션으로 만든다.
3. 팀 소개를 초반부로 옮기고, 유동환/김성균의 배경이 문제 발견과 서비스 탄생으로 이어지게 구성한다.
4. 근거자료는 슬라이드별 링크가 아니라 하나의 evidence index로 통합한다.
5. AC/투자 포트폴리오 기업과 특허 레퍼런스 체크를 새 리서치 태스크로 분리한다.
