# 04_발표준비 읽는 순서

이 폴더는 2026 InBodyLIKE 제출/발표 준비를 위해 만든 실전 자료 묶음이다. 미팅 전에는 아래 순서로 보면 가장 빠르게 큰 그림을 잡을 수 있다.

## 먼저 볼 것

1. `06_지원서_논리개요.html`
   - 브라우저로 열면 지원서 논리, 약점, 출처를 한눈에 본다.
2. `08_몸소_지원서초안.pptx`
   - 실제 제출/발표 초안 덱이다.
3. `09_지원서초안_상세설명.md`
   - 14장 덱의 의도, 말할 때 조심할 표현, 남은 질문을 설명한다.
4. `10_최종정합성검토_20260608.md`
   - 유동환 추가 작업이 기존 momso 준비와 어떻게 맞는지 검증한 정리본이다.

## 미팅 후 반영 자료

- `16_자문미팅_피드백_메모.md`: 2026-06-08 자문 미팅에서 나온 핵심 피드백. 데이터 필연성, 인바디 데이터 필요성, 발표자료 재구성, AC/특허 레퍼런스 체크 과제를 기록한다.

## 근거와 보완 자료

- `07_연희요가위크_도메인근거.md`: 기존 `1,664명` 숫자를 공개 검증값 `1,677명(정산 기준)`으로 교체하는 근거.
- `02_예상_QnA_발표노트.md`: 심사자가 찌를 질문 30개와 방어 답변.
- `05_지원서_생각정리_워크시트.md`: 자문 미팅/지원서 작성을 위한 빈칸형 생각 정리.
- `04_최근Tiro_보완점.md`: 최근 Tiro 원문에서 발표에 반영할 보완점. 원문 보존은 PR #3의 `codex/tiro-archive-raw-preservation` 브랜치에서 보강됐다.
- `03_부록_appendix.pptx`: 발표 후 질문 대비용 부록.

## 덱 재생성

`08_몸소_지원서초안.pptx`는 `build_draft.js`로 다시 만들 수 있다.

```powershell
npm install
node 04_발표준비/build_draft.js
```

주의: 이 폴더의 발표 자료는 private repo 전제다. `tiro-archive/private-raw` 원문을 인용한 자료는 외부 공개 전에 반드시 민감정보 제거 기준을 다시 확인한다.
