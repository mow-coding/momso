## A. 핵심 요약

- 짝꿍 파일: **`08_몸소_지원서초안.pptx`** (14장). 이 09 문서는 *각 슬라이드를 어떻게 말하고, 무엇에 근거하며, 어디서 과장을 조심할지*를 담는다.
- 근거 태그: `[현장근거]`=유동환 현장 감 / `[기술근거]`=김성균 기술·전략 / `[리서치근거]`=공식 통계·웹 / `[가정]`=미검증(PoC로 실측).
- 핵심 포지셔닝: **헤드라인(심사자)과 비전(소비자)을 분리**, 인바디는 헤드라인에서 빼고 "첫 정량 데이터 파트너"로.
- 차별점 순서 = ① HITL+수업맥락(해자) → ② BYOK 데이터주권(게다가). BYOK는 **"네이버·클로드 간편 연동"**으로 부담을 흡수.
- 숫자는 **조건부**로: 기본 20~26억 / 성숙 35~39억 / 공격 60억.

## C. 본문 — 슬라이드별 스크립트·근거·주의

### S1 (표지)
- **메시지:** momso = "수업에서 사라지는 말·교정·감각을, 강사가 검수해 개인 수련 기록으로 남긴다."
- **스크립트:** 자료 없이 10초 안에 이 한 문장 + 데모 1화면.
- **⚠️ 주의:** 인바디를 표지에 넣지 않는다(종속 인상 방지).

### S2 — §0 정체성 (헤드라인 vs 비전)
- **메시지:** 같은 momso를 *심사자에겐 헤드라인*("강사 검수 수련 기록"), *소비자에겐 비전*("내 몸을 가장 잘 아는 세컨드 바디")으로 나눠 말한다.
- **근거:** §0 워크시트 결론 `[현장근거+기술근거]`. 인바디=첫 정량 데이터 파트너(필수 아님).
- **⚠️ 주의:** 두 문장을 한 슬라이드에 섞어 말하면 초점이 흐려진다 — 청중에 따라 *하나만* 앞세운다.

### S3 — §1 문제 (1) 한 통점·두 얼굴
- **메시지:** 통점은 하나("수업의 핵심이 끝나면 휘발"), 얼굴은 둘(수련생 / 강사·운영자).
- **근거:** 빅블루 리뷰 123건(2026-05-08) `[현장+리서치]`, 5/13 기획회의.
- **스크립트:** "수련생은 또 다치고, 강사는 변화를 못 보여줘 재방문을 놓친다."

### S4 — §1 문제 (2) 킬러 인사이트
- **메시지(가장 강한 장면):** "변화는 일어나는데, 강사는 *보여줄* 수 없고 수련생은 *볼* 수 없다 → 동기·재방문 단절."
- **근거:** `[현장근거]` (유동환 현장 관찰). 인바디가 변화를 '보이게', momso가 '의미를' 입힘.
- **⚠️ 주의:** 여성 80~90%는 `[현장 관찰]`로 표기. "살을 뺀다" X → "변화를 함께 본다(참고값)" O.

### S5 — §1 기존 도구 공백
- **메시지:** CRM(예약·결제)·예약앱(접근)·전사앱(글)·측정기(수치)가 각자 일부만 → **수업 맥락 기록 계층이 비어 있다.**
- **근거:** 외부 경쟁/시장 리서치(2026-05-26) `[리서치]`, 노트 06 포지셔닝.
- **스크립트:** 2×2(운영↔수업기록 × 정량↔정성)에서 momso는 우상단 빈 칸.

### S6 — §2 솔루션 흐름 (HITL)
- **메시지:** 수업 전→녹음→AI 초안→**강사 검수**→발행→축적→회수. *AI는 초안만, 강사가 확정.*
- **근거:** PRD v1(2026-06-03) `[기술근거]`, 데이터 3계층(raw→metadata→shareable).
- **스크립트:** "원본 음성·전체 전사는 기본 비공개, 검수본만 발행." (요가 민감성 방어.)

### S7 — §2 차별점 (① 검수+수업맥락 / ② 데이터주권)
- **메시지:** ①(해자) HITL+수업 맥락 기록 — AI 회의록·CRM엔 구조 없음 / ②(게다가) BYOK 비용·데이터 주권.
- **경쟁 5종 한 줄:** "걔네는 X, 우리는 Y + 강사 검수" (바디코디·오붓·Tiro·인바디·포인티).
- **근거:** §2 워크시트 차별점 순서 확정 `[현장+기술]`.
- **⚠️ 주의:** "바디코디가 따라 하면?" → "운영·결제가 본업이라 질적 데이터·요가 검수 구조가 없다."

### S8 — §2 데모
- **메시지:** "이미 만들었다" — 작동하는 모바일 웹앱(강사 5탭·수련생 4탭·발행 게이트). 데모 주인공=강사 AI 위키 / 비전=수련생 세컨드 바디.
- **근거:** InbodylikePrototype.tsx(2026-06-02~03), §2 LLM 주인공 결론.
- **⚠️ 주의:** 세컨드 바디는 "PoC로 검증 중" — *구현 완료로 단정 금지.*

### S9 — §3 시장
- **메시지:** TAM 131,764곳·89.9% 1~4인 / 요가 쐐기(빅블루)→연희동→필라·PT(인바디)→일반.
- **근거:** 공식 통계 `[리서치]`, §3 워크시트(요가 쐐기→필라 확장).
- **⚠️ 주의:** 가정용 인바디 소비자(④)는 *한 줄로만*, 첫 시장과 섞지 않는다. SAM은 `[가정]`. (메인 덱 03 슬라이드13 하위숫자 교체 권고는 `01_리서치_가정채우기_메모` 참조.)

### S10 — §3 BM·단위경제
- **메시지:** B2B 30만(주 매출·재방문 투자) + B2C 7천(바이럴). 곳당 월 41~72만(전환 20~50%). 3년차: 기본 20~26억 / 성숙 35~39억 / 공격 60억.
- **근거:** §3 워크시트. 전환율 50%=동환님 *성숙·완성* 가정 `[현장근거·가정]`. 60억=공격 시나리오.
- **⚠️ 주의:** "60억 근거?" → *조건부터*: "기본 20~26억, 60억은 1,000곳·유료 30명 가정." 회원수·전환율은 `[가정]`, PoC로 실측.
- **숨은 강점:** BYOK로 LLM 추론비를 momso가 안 냄 → 원가↓ `[기술근거]`.

### S11 — §3 BYOK 간편 연동
- **메시지:** "BYOK가 장점"이라고만 쓰면 역효과(자유=부담). **네이버·클로드 '간편 연동'으로 부담을 우리가 흡수.** 발레파킹: 데이터=고객, 운영=momso.
- **근거:** §3 워크시트(Joplin=OneDrive 호환 인사이트) `[현장근거]`, web2.5 가입대행 BM `[기술근거]`.
- **⚠️ 주의:** **"별도 결제 없이"는 스폰서 크레딧 확정 시에만 추가** — *서류 통과 후 운영팀 확인*. 확정 전엔 "간편 연동"까지만.

### S12 — §4 팀
- **메시지:** 요가를 *아는* 유동환 + *만드는* 김성균(50:50). "개발자 없는데 되나" → 작동 데모=증거, 채용은 투자 후. 영업 공백=4중 보완.
- **근거:** §4 워크시트, 영업 채용·투자 원칙(2026-06-05).
- **4중 보완:** 창업자 직접영업 · 연희동 클러스터 등대 · B2C-pull(inbound) · 인바디 파트너 채널 → playbook 검증 후 영업 매니저 채용.
- **⚠️ 주의:** 영업이 가장 약하다는 걸 *정직하게* 인정한 뒤 보완책을 댄다.

### S13 — §5 인바디 협업
- **메시지:** win-win(요가원 진입로↔측정값 맥락화). 3단 연동: 직접입력(즉시)→앱연동(협조)→공식 API(심화). 첫 요청=8~12주 공동 PoC.
- **근거:** 인바디 앱-앱 연동 합의(2026-06-06), 연동 타당성 리서치(2026-05-26).
- **⚠️ 주의:** *"기기 즉시 직결" 약속 금지.* "데이터 바로 연동되나?" → "직접입력·앱연동으로 지금 작동, API 직결은 계약·승인 뒤 단계적."

### S14 — 클로징 / Ask
- **메시지:** 인바디와 8~12주 공동 PoC — 정량×정성 재방문 효과를 빅블루에서 검증. 문제·해결·시장 3줄 요약.
- **스크립트:** "큰 API·계약이 아니라, *작은 PoC*부터." (작은 문 먼저.)

5. **(메타)** 지원서에서 가장 약한 섹션은? 심사자가 가장 의심할 한 가지는?
