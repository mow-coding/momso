## 0. 동기 · 계기 — 왜 이 서비스를 생각했나 (개요에 빠졌던 부분)

momso는 추상적 아이디어가 아니라 **요가원 운영자의 세 가지 현장 체험**에서 나왔다.

1. **'수련 수첩'의 실패.** 유동환은 빅블루 요가에서 수련생 자기이해를 돕는 '수련 수첩'을 도입했으나 실패했다. 수련생은 기록을 부담스러워했고, 운영자는 수십 명 피드백을 손으로 남길 수 없었으며, **수업 중 포착한 생생한 영감이 기록 시점에 휘발**됐다.
   - `[원본: 01_2026_modo_application_original.md · docs/notion-archive/momso-raw-20260526/ · 내용 2026-05-15(지원서) / git 보존 2026-05-26]`
2. **외국인 수련생 AI 리포트 실험(=씨앗).** 수업 후 AI로 정리한 PDF 리포트를 보냈더니 "수업 감각을 즉각 복기해 통증이 호전됐다"는 반응이 왔고, 번역 리포트는 국경을 넘는 연결 도구가 됐다. 이 작은 실험이 momso의 직접적 씨앗이다.
   - `[원본: @2026년 5월 13일 회의 ….md · docs/notion-archive/official-export-20260526-relevant/ · 2026-05-13]`
3. **리뷰가 증명한 통점.** 빅블루 리뷰 123건(3.5년치) 분석에서 고객은 "몸을 다시 배운다 / 통증이 줄었다 / 내 몸의 문제를 알게 됐다"를 반복 언급했다 — '수업의 가치가 사라진다'는 문제의식의 *객관적 근거*.
   - `[원본: @2026년 5월 8일 온라인 창구 리뷰 분석 ….md · docs/notion-archive/official-export-20260526-relevant/ · 2026-05-08]`

> **동기 한 줄:** "수업에서 분명히 생긴 가치가, 끝나면 사라진다. 그걸 붙잡는다."

## 1. 정의 — momso는 무엇인가

- **헤드라인(심사자용):** 수업에서 사라지는 **말·교정·감각을 강사가 검수해 개인 수련 기록으로 남기는** 서비스. "인바디는 몸의 스펙을, momso는 수업의 맥락을 기록한다."
  - `[원본: 20260527_momso_macro_positioning.md · docs/support-programs/inbodylike-2026/research/ · 2026-05-27]`
- **비전 한 줄(동환님 언어):** "그렇게 쌓이면 momso는 내 몸을 가장 잘 아는 '세컨드 바디'가 된다."
  - `[원본: 20260526_momso_deep_research_addendum.md · docs/support-programs/inbodylike-2026/research/ · 2026-05-26]`
- **아닌 것(피할 포지션):** 요가원 CRM / 예약·결제앱 / 웰니스 패스 / AI 회의록앱 / 인바디 장비판매.
  - `[원본: 20260527_momso_macro_positioning.md · …/research/ · 2026-05-27 + outputs/20260527_macro_positioning_gpt_response.md]`

## 2. PROBLEM — 누구의 어떤 문제

- **고객·통점:** 웰니스 강사·운영자와 수련생 모두 — 수업의 핵심(말·교정·감각)이 끝나면 휘발되고, "수련생은 뭘 더 궁금한지 / 지도자는 뭘 더 가르칠지" 소통 장벽이 생긴다. 소규모 사업장이라 운영자가 모든 걸 책임지느라 한 명 한 명을 다 기억 못 한다.
  - `[원본: @2026년 5월 8일 온라인 창구 리뷰 분석 ….md · docs/notion-archive/official-export-20260526-relevant/ · 2026-05-08]`
- **기록 계층의 부재(핵심 프레임):** 요가·웰니스는 비정형·블랙박스라 "기록 계층(record layer)"이 통째로 비어 있다.
  - `[원본: 20260526_lunch_tiro_meeting_notes.md · docs/support-programs/inbodylike-2026/meetings/internal/ · 2026-05-26]`
- **기존 도구의 공백:** CRM(바디코디 등)=예약·결제만 / 예약앱(오붓)=수업 접근만 / 전사앱(Tiro)=음성→글만 / 인바디=몸 수치만. 수업 안의 *언어·피드백·감각*을 기록하는 도구는 없다.
  - `[원본: 20260526_external_competitor_market_research.md · …/research/ · 2026-05-26]`

## 3. SOLUTION — 어떻게 푸나 (HITL 흐름)

- **흐름:** 수업 녹음 → AI 전사·요약 *초안* → **강사 검수**(공유/내부/보류/제외) → 검수된 문장만 개인 리포트로 발행 → 다음 수업 맥락으로 환류.
  - `[원본: 20260603_inbodylike_prd_v1_meeting_grounded.md · docs/planner/briefs/ · 2026-06-03]`
- **HITL(사람 검수)이 핵심:** AI는 초안만, 강사가 확정. 원본 음성·전체 전사는 기본 비공개, 검수본만 공유. 수련생 무단 녹음 금지.
  - `[원본: 20260529_yoodonghwan_domain_concerns_product_strategy.md · …/research/ · 2026-05-29] (원천: 20260529_yoodonghwan_gomin_original.txt · …/source-materials/20260529-tiro-and-domain-notes/ · 2026-05-29)]`
- **데이터 3계층:** raw(원본·비공개) → metadata(초점·태그·RAG) → shareable(강사 확정만).
  - `[원본: 20260603_app_function_bm_valet_summary.md · docs/support-programs/inbodylike-2026/meetings/tiro-archive/ · 2026-06-03]`
- **지금 작동하는 것:** 모바일 웹앱 데모(강사용 5탭·수련생용 4탭, 발행 게이트·면책 구현) — momso.vercel.app.
  - `[원본: InbodylikePrototype.tsx · apps/web/src/prototype/ · git 추가 2026-06-02~03] (빌드로그: 20260602_inbodylike_prototype_build_log.md · docs/planner/codex-sessions/)]`

## 7. 인바디 협업 포인트

- **인바디가 얻는 것:** 미점유 시장인 요가·웰니스 진입 통로. ⚠️ momso는 인바디 없이도 작동 → 인바디는 "전부가 아니라 *첫* 정량 데이터 파트너".

  - `[원본: 20260606_tiro_5JwBLFcHxry69_bassoon_20260606134645_transcript.md · …/tiro-archive/private-raw/ · 2026-06-06 + 20260526_inbody_integration_feasibility_research.md · 2026-05-26]`
