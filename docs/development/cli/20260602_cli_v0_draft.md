# momso CLI v0 Draft

상태: draft
작성일: 2026-06-02

## 목적

PoC 개발자와 운영자가 momso 데모와 데이터 상태를 재현 가능하게 점검할 수 있도록 CLI 후보를 정의한다.

현재는 구현 완료 문서가 아니라 설계 초안이다.

## 명령 후보

### `momso demo status`

현재 데모 상태를 확인한다.

출력 후보:

```text
prototype_url: https://<final-domain>/prototype
build: ok
lint: ok
last_smoke: pending
```

### `momso smoke prototype`

제출용 프로토타입의 5개 화면 이동을 점검한다.

확인 후보:

- 오늘의 수업
- 보호 녹음
- AI 초안 검수
- 수련생 리포트
- 다음 수업 맥락

### `momso class inspect <class-id>`

수업 맥락을 점검한다.

### `momso review list <class-id>`

AI 초안과 강사 검수 상태를 목록으로 본다.

### `momso review set <item-id> --status <status>`

검수 상태를 변경한다.

주의:

- 실제 운영에서는 강사 승인 컨텍스트가 필요하다.
- CLI에서는 PoC/테스트 데이터에만 사용한다.

### `momso report preview <class-id>`

수련생 리포트에 들어갈 항목을 미리 본다.

### `momso report publish --dry-run <class-id>`

발행 가능 여부를 점검한다.

가드:

- 공유 확정 항목이 없으면 실패한다.
- 보류/내부/제외 항목은 발행 대상에서 빠진다.

## 6/12 전 우선순위

CLI 자체 구현보다 아래가 먼저다.

1. Playwright smoke 스크립트
2. Vercel 배포 URL 확정
3. PDF 링크 클릭 검증
4. 제출용 프로토타입 안정화

CLI는 PoC 이후 개발자 경험을 위한 방향성으로 둔다.
