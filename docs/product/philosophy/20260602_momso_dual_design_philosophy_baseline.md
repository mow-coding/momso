# momso Dual Design Philosophy Baseline

상태: draft
작성일: 2026-06-02
작성자: Codex

## 한 줄 요약

momso의 설계 철학은 유동환의 도메인/수업 기록 철학과 김성균의 시스템/데이터 주권 철학을 같은 위계의 입력으로 통합한다.

## 왜 이 문서가 필요한가

6/12 InBodyLIKE 제출용 프로토타입은 단순히 작동하는 화면만 필요한 것이 아니다.

누가 어떤 철학을 제공했고, 그 철학이 제품 구조에 어떻게 반영되었는지 추적 가능해야 한다. 그래야 이후 Claude 리뷰, 발표자료, PoC 설계, 협업자 피드백에서 "왜 이렇게 만들었는가"를 다시 설명할 수 있다.

## 철학 입력 1. 유동환

역할:

- 요가/수업 도메인 지식 제공자
- 수업 현장에서 어떤 기록이 의미 있는지 판단하는 관점 제공
- 지도자와 수련생 사이의 언어, 감각, 교정, 반복 피드백을 놓치지 않게 하는 철학 제공

현재 보존된 원본:

아래 파일은 로컬 작업공간에 보존된 원본이다. GitHub handoff에는 원본 바이너리를 바로 포함하지 않고, 유동환 확인 후 별도 archive 커밋으로 올릴지 결정한다.

```text
docs/product/philosophy/source-archive/20260602_yoodonghwan_docx/Self Optimizing Archiving 모델.docx
docs/product/philosophy/source-archive/20260602_yoodonghwan_docx/Tetris Logic 시스템.docx
docs/product/philosophy/source-archive/20260602_yoodonghwan_docx/Soa 모델의 가장 단순한 핵심에 대해 개념도나 다이어그램으로 표현해줄 수 있어_.docx
```

제품 언어로 번역한 핵심:

- 수업 중 들어오는 말/감각/피드백을 그대로 흘려보내지 않는다.
- 들어온 정보는 바로 공유하지 않고, 검토와 보류를 거친다.
- 의미 있는 기록은 정리되어 다음 수업으로 돌아와야 한다.
- 지도자의 전문성은 AI가 대체하는 것이 아니라, AI 초안을 검수하는 과정에서 드러난다.

momso에 반영된 구조:

- 보호 녹음
- AI 초안
- 요가 용어 확인 필요
- 강사 검수
- 공유/내부/보류/제외 분류
- 다음 수업 맥락 회수

## 철학 입력 2. 김성균

역할:

- 제품 설계 담당자
- 시스템 구조와 Web2.5 / Web3-like 데이터 주권 방향 제공
- "사람과 사람 간의 소통을 안전하고, 진실되고, 지속가능하게 한다"는 팀 비전을 제품 구조로 번역


제품 언어로 번역한 핵심:

- 원본 녹음은 플랫폼이 마음대로 소유하거나 공유하는 콘텐츠가 아니다.
- 원본은 사용자 또는 수업 주체에게 귀속된 source 자료로 본다.
- momso는 동의, 전사, AI 제안, 강사 확정, 리포트 발행, 권한 관리 계층을 맡는다.
- 수련생용 앱은 단순 리포트 뷰어가 아니라, 개인 아카이브/API/관계 기반 공유/SNS형 공유로 확장될 수 있는 공유 계층이다.

momso에 반영된 구조:

- 원본 기본 비공개
- AI 자동 발송 금지
- AI 제안과 강사 확정의 구분
- 검수된 리포트만 발행
- 내 기록 보관
- 외부 연결은 동의 후
- 선발 후 PoC에서 Naver Login, Naver Cloud Object Storage, CLOVA Speech 연동 검증

## 같은 위계로 반영하는 규칙

1. 유동환의 철학은 "도메인 감각"으로만 낮추지 않는다.
2. 김성균의 철학은 "기술 야망"으로만 낮추지 않는다.
3. 제품 결정은 가능하면 두 입력을 함께 만족해야 한다.
4. 한쪽 철학만 반영되는 결정이라면, 그 이유를 문서에 남긴다.
5. 외부 제출자료에는 내부 철학 용어를 직접 노출하지 않고, 심사자가 이해할 수 있는 momso 제품 언어로 번역한다.

## 현재 프로토타입에 들어간 통합 구조

```text
수업에서 발생한 말/감각/몸 데이터
-> 보호 녹음
-> AI 초안
-> AI 제안
-> 강사 확정
-> 검수된 개인 리포트
-> 수련생 개인 아카이브
-> 다음 수업 맥락
```

이 흐름은 유동환의 "수업 기록을 흘려보내지 않고 검토/보류/정리한다"는 관점과, 김성균의 "원본과 검수 기록을 분리하고 사용자 귀속 공유 계층으로 확장한다"는 관점을 동시에 반영한다.

## 제출자료 표현 원칙

InBodyLIKE 제출자료에서는 다음 표현을 사용한다.

- "AI 초안은 자동 공유되지 않습니다."
- "강사가 공유 범위를 확정합니다."
- "원본 녹음과 전체 전사본은 기본 비공개입니다."
- "수련생은 검수된 개인 리포트를 받습니다."
- "선발 후 PoC에서는 사용자별 저장 영역과 외부 연결을 검증합니다."

다음 표현은 피한다.

- "zet"
- "ZET"
- "objet"
- "Web3 SNS를 만든다"
- "각 수련생의 개인 네이버 저장소에 자동 저장된다"
- "CLOVA Note API를 이미 연동했다"

## 아직 남은 과제

- 유동환 원본 docx 3개를 사람이 읽기 쉬운 요약 문서로 별도 정리
- 김성균 Web2.5 방향을 제출 덱용 한 장짜리 MVP/PoC 분리 서사로 정리
- 두 철학이 실제 화면 문구에 어떻게 매핑되는지 체크리스트화
