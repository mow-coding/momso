# momso Product Philosophy

상태: draft
작성일: 2026-06-02

이 폴더는 momso 설계에 반영되는 핵심 철학과 원본 자료를 보존한다.

## 원칙

momso의 제품 철학은 특정 한 사람의 관점만으로 설명하지 않는다.

- 유동환: 요가/수업/도메인 지식 제공자이자 수업 기록 철학 제공자
- 김성균: 제품 설계/시스템 구조/Web2.5 데이터 주권 철학 제공자

두 관점은 같은 위계의 설계 입력으로 다룬다. 한쪽은 "도메인 감각", 다른 한쪽은 "시스템 구조"를 맡지만, 제품 결정에서는 둘 다 근거로 남긴다.

## 원본 보존

유동환이 전달한 docx 원본 3개는 로컬 작업공간의 아래 위치에 보존한다.

다만 GitHub handoff에서는 원본 docx 바이너리를 바로 올리지 않는다. 유동환의 원본 문서이므로, 원격 저장소 포함 여부는 유동환 확인 후 별도 archive 커밋으로 결정한다.

```text
docs/product/philosophy/source-archive/20260602_yoodonghwan_docx/
```

김성균의 Web2.5 / 데이터 주권 방향은 현재 아래 문서에 정리되어 있다.

```text
docs/planner/briefs/20260602_momso_web25_user_owned_recording_strategy.md
docs/planner/codex-sessions/20260602_web25_data_sovereignty_research_log.md
```

## 외부 표현 규칙

내부 사고를 위해 `objet`, `zet`, `ZET`, `Tetris Logic`, `SOA` 같은 표현을 참조할 수 있다.

하지만 InBodyLIKE 제출자료와 심사자용 화면에서는 혼동을 줄 수 있으므로, 외부에는 아래 제품 언어로 풀어 쓴다.

- 원본 자료
- AI 초안
- 강사 검수 기록
- 수련생 공유 계층
- 개인 아카이브
- 관계 기반 공유
- PoC 기술 가설
