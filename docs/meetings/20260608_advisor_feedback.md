# 2026-06-08 · 자문 미팅: InBodyLIKE 지원서·발표자료 논리 보강 피드백

> 한 줄 요약: 자문가가 지적한 핵심은 "왜 momso가 이 데이터 조합, 특히 인바디 데이터를 반드시 써야 하는가"의 필연성이 약하다는 것이며, 발표자료는 목차·논리 지도·근거 인덱스·팀 서사 중심으로 재구성하고 AC/투자 포트폴리오·특허 레퍼런스를 추가 조사하기로 정했다.

- 참여: 유동환 · 김성균 · 자문가
- 맥락: InBodyLIKE 지원서/발표자료를 수정하기 전에, 자문가가 미팅에서 구두로 지적한 논리 구멍과 보강 과제를 정리하기 위함.

## 논의

### 제품/데이터 논리
- **특정 데이터 사용의 필연성이 약하다.** "수업을 녹음해 AI로 정리한다" 수준으로 보이면 단순 전사/회의록 앱과 차이가 흐려진다. momso가 다루는 핵심 데이터가 무엇이고, 음성만으로 부족하다면 무엇이 더 필요한지(강사 검수 데이터, 수련생 감각 메모, 인바디 수치, 수업 이력)를 선명히 보여야 한다.
- 데이터 조합 정리(반영용 초안):

  | 데이터 | 역할 | 단독 사용 시 한계 | momso에서 필요한 이유 |
  |---|---|---|---|
  | 수업 음성/전사 | 수업 중 실제 언어·교정 기록 | 단순 회의록처럼 보일 수 있음 | 지도자의 말·피드백을 포착하는 출발점 |
  | 강사 검수 데이터 | 공유 가능/내부/제외 판단 | 사람 개입 없으면 민감정보·오해 위험 | 요가 도메인에서 안전성·신뢰의 핵심 |
  | 수련생 감각 메모 | 본인이 느낀 변화 | 주관적이고 흩어짐 | 수치로 안 잡히는 웰니스 변화 기록 |
  | 인바디 데이터 | 몸의 정량 참고값 | 수업 맥락 없으면 숫자만 남음 | 수업 전후 변화를 보이게 하는 정량 anchor |
  | 수업 이력/회원별 아카이브 | 장기 변화 흐름 | 개별 수업만 보면 축적 효과 약함 | 다음 수업 준비·재방문 상담의 근거 |

- **"왜 인바디 데이터가 필요한가"가 약하다.** "인바디는 첫 정량 데이터 파트너" 수준에 머물면 심사자가 "인바디 없이도 되는 서비스 아닌가"라고 물을 수 있다. 인바디 데이터를 **수업 맥락 기록이 실제 몸의 변화와 연결되는지 검증하는 정량 anchor**로 재정의해야 한다. 진단/처방이 아니라 수업 언어·감각 변화의 맥락 안에 배치한다는 점을 분명히 한다.
- 핵심 질문을 "인바디와 협업하려면 무엇을 먼저 요청해야 하나"에서 **"인바디 데이터가 빠지면 무엇이 약해지고, 들어오면 어떤 검증이 가능해지는가"**로 바꾼다.

### 발표자료 구성
- **목차와 전체 논리 지도가 필요하다.** 슬라이드별 내용은 있으나 "이 발표가 어디로 가는지"가 한눈에 안 잡힌다. 발표 초반에 목차 → 전체 논리 지도 → 핵심 근거자료 인덱스를 둔다. 논리 흐름: 팀 배경 → 현장에서 발견한 문제 → 왜 수업 데이터인가 → 왜 인바디 데이터인가 → momso의 데이터 조합·HITL → PoC 설계 → BM/시장 확장 → 요청 사항.
- **근거자료는 슬라이드별 하이퍼링크 대신 인덱스 슬라이드로 통합한다.** 슬라이드마다 링크가 붙으면 흐름이 산만해진다. footer 출처 문구는 줄이고, 별도 슬라이드/appendix에 근거 묶음(도메인 실행력, 시장/BM, 경쟁, 인바디 연동, 개인정보/AI, 특허/AC)을 모은다. 특허/AC 레퍼런스는 미수행 상태.
- **팀 소개를 맨 앞으로 배치하고 필모그래픽식 서사로 구성한다.** "우리가 이런 팀입니다"가 아니라 "이런 현장을 겪은 팀이어서 이 문제를 발견했고 그래서 momso가 나왔다"로 보여야 한다. 유동환(빅블루 요가·연희 요가 위크 운영, 현장에서 수업 맥락 소실 문제 경험)과 김성균(기술/데이터 구조 설계, 웹 프로토타입 구현)의 접점이 momso의 출발점임을 드러낸다.

### 레퍼런스 체크 추가 과제
- 시장 규모/경쟁사 관점만으로는 부족하다. (1) AC 산하·투자받은 기업, (2) 헬스케어/웰니스/AI 코칭/스튜디오 SaaS 포트폴리오, (3) momso 기능 정의와 대조되는 특허를 추가로 봐야 한다.
- AC/투자 관점에서 유사 헬스케어 데이터 스타트업의 데이터 필연성 설명 방식, momso와 겹치는 기업, 차별화 포지션을 확인한다. 조사 대상 예: 블루포인트 포트폴리오(헬스케어·웰니스·AI·B2B SaaS), 운동 기록/AI 코칭/건강 데이터/센터 SaaS 기업, 인바디 데이터 활용과 겹치는 스타트업, 국내외 웰니스 데이터 기업.
- 특허는 "낼 수 있나"가 아니라 기존에 어떤 기능·데이터 조합이 주장됐는지, momso가 피하거나 차별화할 표현을 보기 위함. 검색 포인트 초안: 수업 음성 전사·코칭 발화 분석, AI 리포트 human review/instructor approval, exercise/wellness progress report, body composition + coaching recommendation, subjective body sensation self-report, adaptive/longitudinal coaching, user-owned health data·consent-based sharing(BYOK).

## 결정
- 지원서/덱에 **데이터 필연성 섹션**과 **"왜 인바디 데이터인가" 섹션**을 별도로 추가한다.
- 발표 초반에 **목차·전체 논리 지도·근거자료 인덱스** 슬라이드를 둔다.
- **팀 소개를 초반부로 이동**하고 필모그래픽식 서사로 재구성한다.
- 근거자료는 슬라이드별 링크가 아니라 **하나의 evidence index**로 통합한다.
- 핵심 질문을 **"momso가 왜 이 데이터 조합을 반드시 필요로 하며, 그중 인바디 데이터는 어떤 검증을 가능하게 하는가"**로 앞당긴다.

## 아직 미정

- 어떤 AC/VC 포트폴리오 기업이 실제로 momso와 겹치는지, 차별화 포지션의 구체 결론은 미정(리서치 후 확정).
