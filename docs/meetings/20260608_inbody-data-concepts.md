# 2026-06-08 · 인바디 연동·고객 데이터 3소스·디자인 언어 개념 정리

> 한 줄 요약: 6/3~6/7 통화 5건에서 나온 제품/사업 아이디어를 반복 사용 가능한 개념어로 이름 붙이고, 6/12 제출물 반영 우선순위를 정했다.

- 참여: 유동환 · 김성균
- 맥락: 앞선 결정 로그는 6/3 낮 이전 통화 중심이라, 이후 5건의 통화에서 새로 나온 인바디 연동·고객 데이터 구조·디자인 언어·협업 운영 원칙이 정리되지 않은 공백이 있었다. 이를 제품/사업 언어로 묶어 발표·개발에 바로 쓸 수 있게 한다.

## 논의

- **인바디 앱-앱 연동**: momso가 인바디 기기에 직접 붙는다는 뜻이 아니라, 인바디 앱/측정 결과 흐름과 연결된다는 표현. 인바디 가능 센터/공간을 찾는 사용자 흐름 위에 momso가 올라탈 수 있다는 가설도 함께 나옴.
- **고객 데이터 3소스**: 고객 앱에 쌓이는 데이터를 ①인바디 앱 측정, ②선생님앱 지도 데이터, ③고객 직접 입력으로 구분하는 구조. 좌우 균형 같은 인바디 정량 데이터를 요가/자세/정렬 맥락과 연결.
- **타깃 확장 양축**: 좁게는 요가/필라테스 수련자, 넓게는 인바디를 쓰거나 몸에 관심 있는 사람 전체로 확장 가능. 요가 지도자도 자기 몸 관리·수련 기록을 위해 수련생용 경험을 쓸 수 있음.
- **디자인 언어**: AI 제안 영역은 투명/디지털 질감, 인간 검수 영역은 아날로그 질감으로 시각 구분. 살구/노랑/라벤더 키 컬러를 개인/공유/보류 상태에 매핑. 검수 분류(공유·내부·보류)는 단일 선택이 아니라 복수 속성 동시 부여.
- **데이터 주권 흐름**: 강사가 수업 사진을 SNS로 흘리는 대신 momso로 당사자에게 안전 전송·아카이브. 수련생이 공유 허가된 리포트만 자기 LLM/외부 도구에 연결하는 읽기 API(장기 로드맵).
- **PR 영상 반반 구성**: 유동환의 요가 지도자/소통 병목 스토리와 김성균의 기술 설계를 절반씩 담는 홍보 영상.
- **팀·투자 원칙**: 부족한 핵심 역할은 영업이며 투자 유치 이후 채용이 적절. 섣부른 현금 투자보다 인맥 연결·동행·명확한 계약 구조가 중요.
- **협업/아카이브 원칙**: 원본 데이터·인간 메모·시스템 메모를 분리 보존하는 WORM 3계층. GitHub은 원본 파일과 그 추적용 메타데이터를 분리해 추적 가능한 메타데이터를 두는 장부로 본다. AI가 읽고 활용하는 위키/지식 저장소(AI 위키) 개념도 함께.
