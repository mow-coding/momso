# 2026-06-08 · momso 자문가 사전 검수 미팅 (InBodyLIKE 지원 점검)

> 한 줄 요약: 외부 자문가에게 momso 지원 논리를 심사역 관점으로 사전 검수받기 위해, 포지셔닝·BM 숫자·시장 정의·인바디 협업 제안을 어떻게 보강할지 점검했다.

- 참여: 유동환 · 김성균 · 외부 자문가(심사역 관점 게스트)
- 맥락: InBodyLIKE 지원을 앞두고 "말은 좋은데 검증이 약한 아이템"으로 보이지 않으려면 어떤 논리·숫자를 먼저 보강해야 하는지를, 발표 설득이 아니라 약점까지 드러내는 사전 검수로 확인하기 위함.

## 논의

### 한 문장 정의와 포지셔닝
- momso 한 문장: "수업에서 사라지는 말·교정·감각을, 강사가 검수해 개인 수련 기록으로 남기는 서비스." 책임 주체는 AI가 아니라 강사, 결과물은 개인 수련 기록, 1차 도메인은 요가·필라테스·PT 같은 참여형 수업.
- 인바디는 헤드라인에 두지 않고 "첫 정량 데이터 파트너"로 둔다. 인바디를 첫 문장에 넣으면 "인바디 부속 앱/장비 판매"로 오해될 수 있음.
- 심사자용 헤드라인(문제·해결)과 소비자용 비전("내 몸을 가장 잘 아는 세컨드 바디")을 분리. 세컨드 바디는 구현 완료가 아니라 PoC로 검증할 장기 방향으로 표현.

### momso가 아닌 것 (오해 차단)
- 요가원 CRM 아님(예약·결제·회원권은 바디코디·스튜디오메이트·레슨북 등이 강함).
- 예약·결제 앱 아님(오붓, ClassPass류와 정면 경쟁 시 차별점 흐려짐).
- 웰니스 패스·AI 회의록 앱·인바디 장비 판매 모두 아님. 전사는 부품일 뿐 핵심은 강사 검수와 회원별 축적.

### 문제 정의 (한 통점, 두 얼굴)
- 수업의 핵심(말·교정·감각·반응)은 수업이 끝나면 대부분 사라짐.
- 수련생: 일상에서 수업 내용을 다시 못 씀, 같은 부위 재부상, 내 몸 변화 인지 어려움.
- 강사·운영자: 수십 명을 다 기억·관리 불가, 전문성이 기록으로 안 남음, 변화를 못 보여줘 재방문 놓침.
- 사업적 의미: 작은 스튜디오에게 재등록·신뢰 문제. "나아지고 있다"는 느낌이 재방문으로 이어짐.
- 근거: 빅블루 리뷰 123건 분석에서 "몸을 다시 배운다", "통증이 줄었다", "내 몸의 문제를 알게 됐다"는 표현이 반복. 외국인 수련생에게 AI 정리 리포트를 보냈을 때 수업 감각 복기·통증 호전에 도움받았다는 반응이 씨앗.

### 킬러 인사이트
- "변화는 일어나는데, 강사는 보여줄 수 없고 수련생은 볼 수 없다." 인바디는 변화의 일부를 숫자로 보이게 하고, momso는 그 변화가 어떤 수업 맥락·피드백·감각 변화와 함께 나타났는지 설명.
- 표현 주의: "살을 뺀다/체형을 고친다/통증을 치료한다"는 회피, "변화를 함께 본다/참고 지표/맥락 연결"로. 웰니스 유료 고객의 80~90% 여성 관찰은 현장 관찰로만, 공식 통계로 단정 금지.

### 기존 도구의 공백
- 운영 CRM(바디코디)·예약/패스(오붓, ClassPass)·전사(Tiro)·측정기(인바디)·체형분석(Exbody, Bodydot, Bodygram)이 각자 일부만 기록. 수업 안에서 오간 언어·감각을 장기 웰니스 데이터로 남기는 계층이 비어 있음.
- momso 위치: 수업 경험/기록 중심 × 정성 수업 맥락 중심. 인바디 결합 시 정량까지 잇는 하이브리드.

### 해결 흐름 (HITL)
- 수업 전(참고 지표·초점) → 강사 주도 녹음(수련생 무단 녹음 금지) → AI 초안(전사·요약·초점 후보) → 강사 검수(공유/내부 메모/보류/제외) → 발행(검수본만) → 축적(회원별) → 회수(다음 수업 전 맥락 재호출).
- 데이터 3계층: raw(원본 음성·전체 전사, 기본 비공개) / metadata(태그·반복 피드백·초점) / shareable(강사 확정 문장·개인 리포트).
- 안전 설계: 원본 음성·전체 전사 기본 비공개, 검수본만 공유, 공유 확정 항목 없으면 발행 차단, 수련생 앱에 녹음 기능 없음.

### 차별점
- 첫째: HITL + 수업 맥락 기록(요가 수업의 민감성을 이해하고 초점 단위 검수, 검수본만 회원별 장기 축적하는 운영 구조가 해자).
- 둘째: 데이터 주권·비용 구조. 데이터는 고객 소유, momso는 정리·검수·발행·운영 대행. BYOK는 사용자에게 부담이므로 "네이버·클로드 간편 연동"으로 표현, 키 발급 부담은 momso가 흡수.
- 경쟁사 대비: 바디코디=센터 운영 / 오붓=수업 접근성 / Tiro=음성→글 / 인바디=몸의 스펙 / 포인티=PT 운동기록 vs momso=수업 경험·수련 기록·요가 맥락+강사 검수.

### 데모

- PoC 전 프로토타입임을 명확히. "인바디 자동 연동·실전 전사 완성·세컨드 바디 구현 완료"라고 말하지 않음.
- 데모 주인공은 강사 흐름(돈 내는 1차 고객이 원장/강사).

### 시장
- 층 구조: 초기 쐐기(빅블루 요가·연희동 1:1/소그룹) → 1차 시장(요가+필라테스+PT 프리미엄 참여형 스튜디오) → 확장(인바디 보유 필라/PT, 골프·테니스·재활) → 장기(가정용 인바디·웨어러블·홈 웰니스).
- 대리 지표: 스포츠산업 사업체 131,764개 / 매출 84.7조 원 / 스포츠시설업 49,082개 / 기타 스포츠 교육기관 20,507개 / 체력단련장업 15,548개 / 체육교습업 2,999개 / 1~4인 소규모 비중 89.9%. 매크로 숫자를 직접 TAM으로 쓰면 과대.
- 전략: 가장 까다로운 요가에서 인바디 없이도 통하게 만들고, 인바디 깔린 필라/PT로 넘어가 협업 확대.

### BM
- B2B2C 혼합형. B2B 스튜디오 구독 월 30만 원 중심(주 매출) + B2C 개인 기록권 월 5천~7천 원.
- 월 30만 원 방어: 단순 SaaS(국내 예약·회원관리 월 15만~22만 원)로 보이면 안 되고, 수업 언어 기록·감각 데이터·월간 리포트·경량 장비·초기 세팅을 포함한 프리미엄 데이터 패키지로 설명.
- 패키지: Basic 9.9만~14.9만 / Core 19.9만~24.9만 / Premium 29.9만~39.9만(30만 원 방어) / Premium Plus 49.9만 이상.
- B2C 월 7천 원: 단순 저장(톡클라우드·iCloud·Google One)보다 비싸 보이나 "개인 수련 기록권"으로는 방어 가능.
- 매출식: 연매출 = 업장 수 × [B2B 월 구독료 + 업장당 B2C 유료 수련생 수 × 7,000원] × 12개월.
- 업장당 민감도(B2B 30만 기준): 유료 5명→연 402만 / 10명→444만 / 20명→528만 / 30명→612만.
- 3년차 시나리오: 보수(300개·B2B 10만·10명)=6.12억 / 기준(500개·20만·20명)=20.4억 / 프리미엄 기준(500개·30만·20명)=26.4억 / 공격(1,000개·30만·20명)=52.8억 / 60억 달성(1,000개·30만·30명)=61.2억.
- 화법: 본문은 3년차 500개 업장·20.4억~26.4억 먼저, 60억은 공격 시나리오 조건부로만. 유료 전환율 50%는 성숙 가정. 회원 수·전환율·CAC·해지율은 아직 가정이며 PoC 실측 필요.

### 데이터 주권
- "BYOK가 장점"은 원장에게 부담. 네이버 오브젝트 스토리지·앤트로픽 Claude "간편 연동"으로 표현하고 연결/설정/운영 부담은 momso가 흡수.
- 발레파킹 비유: 차(데이터)는 고객 소유, momso는 운전·정리(기록 정리·검수·발행·운영) 대행, 주차장은 차를 소유하지 않음(momso는 데이터를 팔거나 가두지 않음).
- "별도 결제 없이"는 스폰서 크레딧 확정 후에만. InBodyLIKE 네이버/Claude 크레딧은 공식 프로그램 자료에 있으나 발표 전 운영팀 재확인 권장.

### 인바디 협업
- 인바디가 얻는 것: 측정값의 "그래서 뭐?" 문제를 수업 언어·리포트로 해소, 요가·프리미엄 웰니스 도메인 진입 통로, 강사 검수형 정성 데이터 결합.
- momso가 얻는 것: 정량 참고값, 브랜드 신뢰, 리포트 가치 검증, B2B 영업 채널.
- 연동 로드맵: 1단계 직접 입력·결과지 사진/PDF 업로드(계약 없이 즉시 PoC) → 2단계 InBody App/Lookin'Body export(CSV/Excel) → 3단계 Web API/OAuth(유료 구독·신청·승인·계약 필요).
- 금지 표현: "인바디 API 즉시 연동" / "모든 지표 바로 가져옴" / "요가로 체성분 개선" / "인바디 장비를 30만 원 패키지 포함".
- 첫 요청: 큰 API 계약 대신 8~12주 공동 PoC. 시작 업장 빅블루 요가 1곳, 확장 후보 연희동 프리미엄 요가/필라 3~10곳, 업장당 10~30명. 산출물: 개인 월간 수련 리포트·강사용 피드백 요약·업장 리텐션 리포트.

### 약점 W1~W6
- W1 첫 시장 정의: 요가는 도메인 강점이나 인바디 보유율은 필라/PT가 높을 수 있음 → 요가 쐐기/필라·PT 확장 분리.
- W2 인바디 연동: 공식 API는 계약·승인 필요, 즉시 자동 연동처럼 말하면 위험 → 수동 입력/업로드형 PoC.
- W3 데모 범위: 실서비스 데이터 연동 완료 아님 → PoC 전 데모로 표기.
- W4 BM 숫자: 전환율·업장 수·CAC·해지율 실측 전 → 조건 명시 + PoC KPI 실측.
- W5 영업 공백: 2인이 영업/CS/개발 동시 감당 어려움 → founder-led sales 후 투자 뒤 영업 매니저 채용.
- W6 개인정보/AI 신뢰: 명시 동의·원본 비공개·강사 검수·삭제권 설계.
