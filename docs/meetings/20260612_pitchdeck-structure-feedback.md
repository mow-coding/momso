# 2026-06-12 · InBodyLike 1차 제출 피치덱 구성 논의

> 한 줄 요약: 1차 제출 덱은 "슬라이드 수 최소·정보 집약" 기조를 유지하되, 첫 페이지 UI를 리포트 공유 미리보기와 수련생 홈 화면으로 교체하고, 마지막에 팀 마일스톤 4단계 슬라이드를 하나 추가하기로 했다.

- 참여: 유동환 · 김성균 · 게스트 친구(외부 피드백)
- 맥락: InBodyLike 2026 1차 서류(발표 자료 겸 사업 계획서, 6월 12일 마감) 제출을 앞두고, 외부 시선의 피드백을 받아 덱의 최종 구성을 확정하기 위해 모였다.

## 논의
- AI 구현 세부(용어 딕셔너리 구조, 소스 3개의 구성 요소, RAG 방식 선택 근거)를 덱에 담을지. 게스트 친구가 "녹음 요약 앱과 달라 보이지 않는다"며 요가 도메인을 반영한 전사 방식이나 용어 딕셔너리 예시를 'for example' 형태로라도 보여주자고 제기했다. 김성균은 해당 내용이 아직 구현 전 단계이고, 이전 미팅에서 받은 "업계 사람이 보면 바로 알 수 있게 간결하게" 피드백을 수용한 방향이라는 이유로 반대했다.
- 발표 전략의 관점 차이. 김성균은 "궁금하게 만들면 성공", 유동환은 "의지와 구체적 계획을 보여줘야 신뢰" 입장이었다. 유동환은 구체 예시로 빅블루 요가의 '정렬 에센스'·'홈 리커버리' 수업 데이터를 클로바 노트로 전사해 패턴을 키워드로 정립하는 계획을 들 수 있다고 했다. 논의는 현 단계 덱이 보여줄 것은 역량 과시보다 시장 위치 파악과 전략적 사고라는 쪽으로 정리됐다. 아이템·아이디어·역량에 과도하게 치중한 발표는 부작용이 많다는 창업학 강의 내용이 근거로 나왔다.
- 마지막 슬라이드에 팀원 역할 분담·역량을 명시하자는 유동환 제안. 김성균은 구상한 지 한 달도 안 된 팀이 역할을 명시하면 "역할놀이처럼 보일 수 있다"고 반론했고, 유동환이 제안을 철회했다.
- 마일스톤 vs 공동 PoC 제안. 통상 사업계획서 말미에는 마일스톤이 들어가지만, 이 프로그램은 4개월간 PoC를 함께 수행할 파트너를 선발하는 자리라서 공동 PoC 제안(우리가 할 수 있는 것 + 파트너에게 돌아가는 이익)을 마일스톤 대신 배치한 현행 구성이 맞다고 확인했다.
- PoC 제안 슬라이드(Commitment/Ask)를 2장으로 나누자는 게스트 친구 제안. 심사가 인쇄본을 넘기며 읽는 방식이라 슬라이드 수를 최소화한다는 기조로 1장을 유지하기로 했다.
- 가시성 vs 정보 집약성. 이 자료는 발표 자료이자 사업 계획서의 이중 역할이라, 현재 우선순위는 정보 집약이고 가시성(폰트 통일·자간·색상 일관성)은 균형 수준에서 처리하기로 정리했다.

## 결정
- 1차 제출(6월 12일 마감)은 현행 슬라이드 구성을 유지하되, 첫 페이지 UI 화면을 리포트 공유 미리보기 + 수련생 홈 화면(최근 몸소 리포트 요약, 하타 베이직·인바디 참고 수업 목록)으로 교체한다.
- 기술 세부(용어 딕셔너리 구조·소스 구성·RAG 설계)는 1차 자료에 넣지 않고, 면접(2차) 단계에서 구두로 구체적으로 설명할 수 있게 준비한다.
- 팀원 역할 분담 명시 슬라이드는 넣지 않는다(제안자 유동환이 철회).
- PoC 제안 슬라이드는 1장 유지(2장 분리는 채택하지 않음).
- 마일스톤 대신 공동 PoC 제안을 배치한 현행 구성 유지.
- 마지막 슬라이드 1장 추가(회의 말미 최종 결정): 주관 AC 블루포인트파트너스에게 팀의 잠재력을 보여주는 마일스톤 4단계(① 요가원 팀으로 시작 ② InBodyLike(현재) ③·④ 후속 단계) 방향. 역량 과시·역할 분담 형식이 아니라 단계별 증명 축으로 설계한다.

## 아직 미정

- 마일스톤과 팀원 기여 영역을 함께 시각화하는 슬라이드는 프로젝트가 충분히 축적된 시점에 재검토(장기 과제).
