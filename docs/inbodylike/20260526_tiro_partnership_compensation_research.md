## 1. 한 페이지 결론 요약

### 1-1. 핵심 결론

금요일 Tiro 미팅에서 가장 안전한 포지션은 `외주 개발`도, `지분 파트너`도 아니라 **제한적 기술 PoC 파트너**다.

Tiro는 실시간 전사, 맥락 기반 요약, 도메인 전문용어 인식, 커스텀 템플릿, 팀 공유, 엔터프라이즈 API/Webhook, API/MCP/CLI 베타 기능을 제공한다. 따라서 몸소가 처음부터 자체 STT/요약 엔진을 만들기보다, **Tiro의 전사/요약/템플릿 기능을 요가·웰니스 수업 기록 도메인에 적용해 검증하는 구조**가 자연스럽다.

### 1-2. 몸소의 권장 협상 방향

| 구분 | 권장 결론 |
|---|---|
| 첫 미팅 | "Tiro 기술이 요가 수업 환경에서도 작동하는지 검증하고 싶다"는 탐색 미팅으로 열기 |
| 1차 보상 | 무료/할인 테스트 크레딧 또는 소액 유료 테스트. 지분, 워런트, 매출쉐어는 먼저 꺼내지 않기 |
| InBodyLIKE 서류 | Tiro의 서면 동의가 있을 때만 "PoC 협의 중인 AI 전사/요약 기술 파트너 후보"라고 표현 |
| 공모전 선정 후 | 8~12주 유료 PoC. API/Export, 데이터 처리 계약, 삭제/반출 조건, 성과지표 명시 |
| 장기 협력 | 단순 API 제공이면 사용료 계약. 고객 소개/공동영업까지 하면 특정 고객·기간 한정 매출쉐어 |
| 지분/워런트 | 원칙적으로 보류. 제품 설계, 도메인 튜닝, 고객 확보, 장기 인프라화에 실질 기여할 때만 마일스톤형 워런트 검토 |

### 1-3. 절대 먼저 약속하면 안 되는 것

아래 표현과 약속은 피해야 한다.

- 지분 제공
- 투자금 일부 제공
- 전사/요약 독점권
- 전체 회사 매출쉐어
- "투자받으면 몇 퍼센트 드리겠다"는 표현
- 공식 파트너 표현
- 로고 사용

advisor equity의 글로벌 벤치마크를 보면 pre-seed advisor 중앙값은 약 0.21% 수준이고, 1% 이상을 받은 경우는 일부에 그친다. Founder Institute의 FAST Agreement도 advisor equity는 전략적 조언자용이지, 전통적인 프로젝트 컨설팅이나 work-for-hire 관계용이 아니라고 설명한다.

### 1-4. Tiro에게 제안할 수 있는 혜택

처음에는 다음 정도가 적절하다.

- 무료/할인 PoC 대가로 제한적 피드백 제공
- 공동 사례화 가능성
- 발표자료 내 기술 파트너 후보 언급, 단 서면 동의 후
- 공모전 선정 시 유료 PoC 전환
- 장기 유료 고객화 가능성

Tiro는 이미 개인 Lite $7, Pro $13, Max $29, 팀 플랜 계정당 $29, 엔터프라이즈 별도 문의 가격을 공개하고 있고, API/MCP/CLI는 베타로 안내되어 있다. 따라서 몸소는 "무상 개발"이 아니라 **공식 플랜/API를 기반으로 한 소규모 실험**부터 제안하는 것이 좋다.

해석/권장:

- Tiro와 몸소의 1차 구조는 **단순 SaaS 사용 + API 기반 PoC 파트너십**이다.
- 공동개발이나 지분형 전략 제휴는 Tiro가 단순 전사 도구를 넘어 요가 도메인 템플릿 설계, 제품 로드맵, 고객 확보까지 실질 기여할 때만 다음 단계로 검토한다.
- 공동개발 계약에서는 기존 IP와 새로 생긴 IP, 즉 background IP와 foreground IP를 명확히 정의해야 한다.
- 공동소유는 상업화, 라이선스, 분쟁을 복잡하게 만들 수 있으므로 초기에는 피하는 편이 안전하다.

## 4. 지분/워런트/옵션 기준

### 4-1. 기본 판단

외부 기술 파트너에게 지분을 주는 것은 일반적인 1차 선택지가 아니다.

Tiro가 이미 만든 SaaS/API를 제공하는 수준이라면 보상은 다음으로 처리한다.

- 사용료
- 크레딧
- 공동 사례화
- 제한적 매출쉐어

지분은 **개인이 advisor로 정기 기여하거나, 파트너사가 고객·기술·제품 설계에 장기적·핵심적으로 기여할 때만** 검토한다.

### 4-2. 기여 수준별 지분/워런트 판단

| 수준 | 고려 가능한 지분/워런트 | 기여 수준 | 몸소 적용 판단 |
|---|---:|---|---|
| 단순 API/SaaS 제공 | 0% | Tiro 기능 사용, 기술지원 제한적 | 지분 금지. 사용료로 처리 |
| 샘플 테스트 협조 | 0% | 1~3개 수업 테스트, 크레딧 제공 | 사례화·유료 전환권 정도 |
| 기술 자문 | 0.05~0.1% | 월 1~2시간, STT 품질·AI 요약 조언 | 개인 advisor일 때만 가능 |
| 표준 advisor | 0.1~0.25% | 월 2~4시간, 제품 방향·기술 선택·소개 일부 | Carta median 0.21%와 유사한 하한 구간 |
| 전략 advisor | 0.25~0.5% | 정기 미팅, 요가/AI 도메인 설계, 고객/투자자 소개 | 개인이 명확히 관여할 때 |
| 핵심 전략 파트너 | 0.5~1% | 제품 설계, 도메인 튜닝, PoC 운영, 고객 확보에 깊게 관여 | 워런트형·마일스톤형만 검토 |
| 공동창업자급 기여 | 2% 이상 | 핵심 제품을 함께 만들고, 장기 운영·리스크 부담 | Tiro B2B 미팅 단계에서는 해당 없음 |
| 사실상 CTO/공동창업자 | 5% 이상 | 전업 또는 준전업, 핵심 기술·팀·제품 책임 | 현재 Tiro 협업 목적과 다름 |

### 4-3. 지분율별 의미

| 조항 | 권장 구조 |
|---|---|
| 베스팅 | 12~24개월 월별 베스팅. advisor는 3개월 cliff 가능 |
| 마일스톤 | 샘플 테스트 완료, InBodyLIKE PoC 선정, 유료 업장 10곳 전환, API 안정 연동 등 객관식 |
| 계약 종료 | 미베스팅분 자동 소멸 |
| 권리 행사 | 워런트는 행사기간·행사가격·희석효과·종료 후 권리를 명확히 |
| IP | Tiro 기존 기술은 Tiro 소유, 몸소 리포트 구조·템플릿·서비스 기획은 몸소 소유 |

| 독점 금지 | Tiro 사용 의무, 몸소의 다른 STT/AI 업체 사용 금지 조항은 넣지 않음 |

### 4-5. "투자받으면 몇 퍼센트 떼준다"가 위험한 이유

이 표현은 피해야 한다.

- 가치평가가 없다.
- 투자 전후 기업가치, fully diluted 기준, 옵션풀 반영 여부가 불명확하다.
- 미래 투자자 실사에서 cap table 리스크로 보일 수 있다.
- Tiro가 무엇을 해야 권리가 생기는지 성과 조건이 불명확하다.
- 증권·세무·상법 이슈가 생길 수 있다.

대안:

- "향후 실제 투자 의향이 있다면 정식 투자 라운드에서 조건을 협의한다."
- "특정 마일스톤 달성 시 소액 워런트를 검토한다."

## 5. 매출쉐어/리셀러 기준

### 5-1. 일반 벤치마크

B2B SaaS에서 리퍼럴·공동영업 수수료는 파트너의 기여도에 따라 달라진다.

| 상황 | 권장 범위 | 적용 대상 | 몸소 적용 |
|---|---:|---|---|
| Tiro가 기술만 제공 | 0% revenue share | 사용료/API 비용만 | 매출쉐어 없음 |
| Tiro가 고객을 소개만 함 | 5~10% | 해당 고객의 첫 12개월 B2B 순매출 | Tiro 소개 업장 1곳 월 30만 원이면 첫해 18만~36만 원 수준 |
| Tiro가 구매 의향 높은 고객을 연결 | 10~15% | 해당 고객 첫 12개월 순매출 | deal registration 필요 |
| Tiro가 공동영업·데모·계약 성사에 관여 | 15~25% | 해당 고객 첫 12개월 순매출 | 고객별·기간별로만 |
| Tiro가 리셀러로 판매·청구·CS 일부 담당 | 20~30% margin | Tiro가 판매한 패키지 | 초기에는 비추천 |
| Tiro 기능 기반 유료 모듈 | 10~20% | Tiro-powered 기록 모듈 순매출 | 전체 몸소 매출이 아니라 기능 매출만 |
| Tiro가 핵심 인프라와 고객 확보 모두 기여 | 20% 내외 + 최소 사용료 | 특정 기간·고객군 | Stage 4 이후 협의 |

### 5-2. 전체 회사 매출쉐어가 위험한 이유

전체 회사 매출쉐어는 피해야 한다.

몸소의 매출에는 B2B 구독료, B2C 기록권, 인바디 연동, 리포트, 현장 세팅, 장비, 향후 PT·골프·테니스 확장까지 포함될 수 있다. Tiro가 기여한 것은 음성 전사·요약 또는 일부 고객 소개인데, 전체 매출에 권리를 걸면 후속 투자 실사에서 외부 파트너가 회사 수익에 포괄 권리를 가진 구조로 보일 수 있다.

### 5-3. 안전한 매출쉐어 설계

| 설계 항목 | 안전한 문구 |
|---|---|
| 범위 | Tiro가 서면 등록한 특정 고객 또는 Tiro API가 사용된 특정 기능 매출 |
| 기간 | 계약 체결 후 12개월, 길어도 24개월 |
| 기준 | gross revenue가 아니라 환불·세금·결제수수료·장비비 제외 후 net revenue |
| 제외 | 장비비, 설치비, 인바디 연동비, 몸소가 자체 확보한 고객, 기존 고객, B2C 전체 매출 |
| 지급 조건 | 고객이 실제 결제하고 환불기간이 지난 후 |
| 상한 | 고객당 또는 월별 cap 설정 |
| 종료 | 계약 종료 후 신규 고객에 대한 권리 없음 |
| 추적 | deal registration, CRM 기록, 이메일 기준 |

### 5-4. 몸소 가격모델에 붙이는 안전안

| 몸소 모델 | 위험한 방식 | 안전한 방식 |
|---|---|---|
| B2B 월 30만 원 패키지 | 전체 월 30만 원의 20%를 Tiro에게 계속 지급 | Tiro가 소개한 업장에 한해 첫 12개월 5~15%, 또는 Tiro 전사 모듈 원가+마진 지급 |
| B2C 월 7,000원 | 모든 유료 수련생 매출의 10% 지급 | Tiro 기능이 포함된 프리미엄 기록 리포트에 한해 1인당 500~1,000원 또는 특정 기능 순매출 10% |
| 공모전 PoC | 선정되면 매출쉐어 약속 | 선정 후 별도 유료 PoC 계약 체결 |
| 장기 API 사용 | 영구 매출쉐어 | 사용량 기반 API/SaaS 비용 또는 연간 계약 |

## 6. 몸소에 맞는 단계별 권장안

### Stage 0. 금요일 첫 미팅

| 항목 | 권장안 |
|---|---|
| 확인할 것 | 요가 수업 녹음에서 전사 정확도, 음악/호흡/작은 목소리 대응, 화자 분리, 전문용어 사전, 커스텀 템플릿, API/Export, 비용, 데이터 삭제, 학습 미사용, 엔터프라이즈 계약 가능성 |
| 절대 약속하면 안 되는 것 | 지분, 워런트, 투자금 일부, 독점권, 전체 매출쉐어, 공식 파트너 표현, 로고 사용 |
| 말해도 되는 혜택 | 공모전 선정 시 유료 PoC, 공동 사례화 가능성, 발표자료 내 기술 파트너 후보/협의 중 표현 가능성, 장기 고객화 가능성 |

추가 확인:

- Tiro 개인정보처리방침은 음성·영상 데이터가 서비스 운영 목적으로 AssemblyAI, OpenAI, AWS 등 국외 처리자에게 이전될 수 있다고 설명한다.
- 음성 데이터는 처리 후 즉시 파기하고 AI 학습·모델 개선·마케팅에 사용하지 않는다고 안내한다.
- 몸소는 수업 참여자 동의문에 외부 AI 처리 및 국외 이전 가능성을 반영해야 한다.

### Stage 1. 샘플 테스트

| 항목 | 권장안 |
|---|---|
| 샘플 | 20분, 60분, 90분 요가 수업 각 1개 또는 총 1~3개 |
| 데이터 | 가능하면 실제 수업 전에는 가명/시연 수업으로 시작 |

| 비용 | 무료 체험 300분 또는 별도 테스트 크레딧 요청. 안 되면 소액 유료 사용 |
| 산출물 | 수업 전사문, 지도자 피드백 추출, 수련생 감각 질문 후보, 월간 리포트 초안 |

> 지금 InBodyLIKE 공모전에 지원하려고 하는데, 핵심 기능 중 하나가 수업 현장의 음성을 전사하고 요약해 수업 맥락을 구조화하는 것입니다. 이 기능을 처음부터 자체 개발하기보다, Tiro의 전사·요약·커스텀 템플릿 기술이 요가 수업 환경에서도 잘 작동하는지 제한적으로 검증해보고 싶습니다. 오늘은 외주 개발 의뢰보다는, Tiro를 기술 PoC 파트너 후보로 보고 어떤 방식의 테스트와 협업이 가능한지 논의드리고 싶습니다.

### 8-2. Tiro에게 제안할 협업 문장

> 저희가 원하는 첫 단계는 큰 개발 계약이 아니라, 요가 수업 녹음 1~3개를 대상으로 Tiro의 전사 정확도, 요가 용어 인식, 문단 요약, 커스텀 템플릿 가능성을 검증하는 작은 PoC입니다. 결과가 좋으면 InBodyLIKE 선정 후 유료 PoC와 공동 사례화까지 확장할 수 있습니다.

### 8-3. 우리가 얻고 싶은 것

> 저희가 이번 미팅에서 확인하고 싶은 것은 네 가지입니다. 첫째, 실제 요가 수업처럼 음악·호흡·움직임 소음이 있는 환경에서 전사 품질이 어느 정도인지. 둘째, 수업 리포트용 커스텀 템플릿을 만들 수 있는지. 셋째, API나 Export 방식으로 몸소 리포트에 결과를 연결할 수 있는지. 넷째, 수업 음성·전사문·요약문을 다룰 때 데이터 보관, 삭제, 학습 미사용, 외부 처리 고지를 어떻게 설계해야 하는지입니다.

### 8-4. Tiro가 얻을 수 있는 것

> Tiro 입장에서는 회의록을 넘어 요가·웰니스 수업이라는 새로운 vertical use case를 검증할 수 있습니다. 공모전 선정 후에는 Tiro가 "전사·요약 기술 파트너"로 사례화될 수 있고, 향후 프리미엄 요가·필라테스·PT 스튜디오 시장으로 확장할 때 유료 사용, 공동 사례, 공동영업 가능성도 생깁니다. 다만 초기에는 과장된 파트너십 표현보다 작은 테스트에서 명확한 성과를 만드는 것이 양쪽 모두에게 안전하다고 봅니다.

### 8-5. 지분 이야기가 나왔을 때 안전하게 답하는 문장

> 지분이나 워런트는 저희도 매우 신중하게 보고 있습니다. 초기에는 Tiro를 유료 SaaS/API 파트너로 사용하는 구조가 맞다고 생각합니다. 다만 Tiro가 단순 API 제공을 넘어 제품 설계, 요가 도메인 템플릿, PoC 운영, 고객 확보까지 장기적으로 깊게 관여하게 된다면, 그때 별도 마일스톤 기반 워런트나 전략적 보상 구조를 검토할 수 있습니다. 오늘 단계에서는 지분 약속보다는 테스트 범위와 데이터·기술 조건을 먼저 정리하고 싶습니다.

### 8-6. 무료 PoC 이야기를 꺼낼 때 부담스럽지 않게 말하는 문장

> 초기에는 저희도 아직 공모전 제출 전이라 예산과 범위를 크게 잡기는 어렵습니다. Tiro에 이미 무료 체험이나 크레딧 구조가 있는 것으로 알고 있는데, 요가 수업 샘플 1~3개 정도를 테스트할 수 있는 제한적 크레딧이나 할인 테스트가 가능할까요? 결과가 유의미하면 선정 후에는 유료 PoC로 전환하는 방향을 열어두고 싶습니다.

### 8-7. 공모전 자료 언급 가능 여부 확인 문장
