# docs 안내

이 폴더는 `momso` 프로젝트 문서를 모아두는 곳입니다. 문서 성격이 섞이면 나중에 김성균과 유동환이 서로 파일을 찾기 어려워지므로, 아래 기준으로 저장 위치를 나눕니다.

## 폴더 구분

| 위치 | 용도 |
|---|---|
| `support-programs/` | 지원사업, 공모전, 오픈이노베이션 자료실 |
| `planner/` | 제품 기획, 사용자 흐름, figma 전달서, codex 세션 요약 |
| `product/` | 합의된 제품 방향, 현재 제품 기준, 보존 중인 과거 제품 기획 |
| `development/` | 개발, 인프라, 실행 방법, 구현 로그 |

## 기본 원칙

- 지원사업 제출 여부, 문의 내역, 평가 일정, 리서치, 회의록은 `support-programs/`에 둡니다.
- 지원사업 회의록은 `support-programs/{사업명}/meetings/`에 둡니다.
- 회의 중 나온 행정 액션은 회의록에만 묻어두지 말고, 필요하면 같은 사업 폴더의 `admin/`에도 짧게 다시 기록합니다.
- 제품 기능 아이디어나 화면 기획은 `planner/`에 둡니다.
- 합의된 제품 기준은 `product/`에 둡니다.
- 개발 실행 방법과 인프라 기록은 `development/`에 둡니다.
- 전화번호, 주민등록번호, 계좌번호, 신분증, 서명 파일처럼 민감한 개인정보는 저장소에 반복 기재하지 않습니다.

## 파일 이름 규칙

가능하면 날짜와 짧은 설명을 함께 씁니다.

```text
2026-05-26-inbodylike-representative-attendance.md
2026-05-26-lunch-tiro-meeting-notes.md
2026-05-27-submission-checklist.md
```
