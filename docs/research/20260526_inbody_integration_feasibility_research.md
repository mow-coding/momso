# 인바디 연동·제품·PoC 현실성 — 몸소는 장비 판매가 아니라 인바디 데이터를 수업 맥락과 잇는 기록·리포트 레이어로 가야 한다

> TL;DR: 전문 인바디 장비는 렌탈비만 월 11.6만~93.8만 원이라 월 30만 원 패키지에 넣기 어렵다(270S만 제한적 가능). 인바디는 이미 장비·앱·클라우드·API 생태계를 갖췄으므로 몸소가 새로 할 일은 장비 유통이 아니라 인바디 수치를 수업 언어·피드백·감각 기록과 연결하는 일이다. PoC는 장비 없이 결과지 업로드·수동 입력만으로도 시작할 수 있다.

- 작성일: 2026-05-26
- 질문: 몸소가 인바디와 실제로 어떻게 결합될 수 있는가 — 장비 도입비, 데이터 연동 경로, 요가원 PoC 현실성, 리스크를 구체적으로 확인한다. (기존 BM·시장규모 조사는 반복하지 않고 그 위에 기술 원리·제품 라인업·가격·연동·PoC·리스크를 얹는다.)

## 배경

기존 결론인 **"B2B 월 30만 원은 단순 SaaS가 아니라 수업 기록·리포트·경량 장비·세팅 패키지일 때만 방어 가능"**이라는 전제를 유지하되, 이번에는 "몸소가 인바디와 실제로 어떻게 결합될 수 있는가"에 집중했다.

핵심 질문은 단순하다. 요가·필라테스 스튜디오가 인바디 장비를 직접 사거나 빌려 쓰는 게 현실적인가, 아니면 몸소는 다른 자리에 서야 하는가. 결론은 후자다. 몸소는 인바디 장비를 직접 판매·렌탈하는 서비스가 아니라, **인바디 데이터를 요가·필라테스·PT 수업의 언어·피드백·감각 기록과 연결하는 기록·리포트 레이어**로 포지셔닝해야 한다.

## 방법

- 인바디·InBodyLIKE 공식 자료(기술 설명, 제품 라인업 페이지, 데이터 연동 페이지)에서 기술 원리·제품 사양·연동 방식을 확인했다.
- 공식 판매가는 대부분 "구매 문의" 방식으로 비공개이므로, 제3자 렌탈몰·중고 마켓플레이스·가격비교 사이트를 **대리 지표**로 사용했다(공식 가격이 아님을 명시).
- 개인정보보호법·생성형 AI 개인정보 처리 안내서를 참고해 데이터 처리 리스크를 정리했다.
- 본 자료는 외부 리서치를 정리·재구성한 것으로, 출처 원문 전체를 별도로 재검증한 것은 아니다. 각 항목은 출처가 있는 `사실`과 발표·사업화 판단인 `해석/권장`으로 구분했다.

## 발견

### 1) 인바디 기술 원리는 요가·필라테스 언어와 연결될 수 있다

- **BIA(생체전기임피던스 분석):** 몸에 약한 전류를 흘려 저항값(임피던스)으로 체성분을 추정한다. 인바디는 이를 부위별·다주파 방식으로 고도화한 Direct Segmental Multi-Frequency BIA를 강조한다.
- **부위별 직접 측정:** 오른팔·왼팔·몸통·오른다리·왼다리 5개 부위를 독립 측정한다. "오른쪽 골반이 더 닫혀 있다", "왼발 접지가 약하다" 같은 수업 언어와 연결 가능하다.
- **다주파 측정:** 세포외수분(ECW)과 세포내수분(ICW)을 구분한다. 요가 맥락에서는 회복 컨디션·수련 전후 몸 상태의 보조 지표로 쓸 수 있으나 "부종 개선" 같은 의료 표현은 위험하다.
- **8점 접촉 전극:** 손·발 모두 잡고 측정해 좌우·상하 균형을 본다. InBody270S는 4극 8점 터치식 전극법을 명시한다.

권장 표현: 인바디 지표는 평가·진단이 아니라 **수련 경험을 더 잘 이해하기 위한 정량 참고값**으로 쓴다(체중→"감량 성공/실패" 금지, 위상각→"건강 나이 진단" 금지 등).

### 3) 데이터 연동은 비API 방식부터 시작할 수 있다

인바디는 Lookin'Body Web, LB120, Web API, OAuth API, Open Protocol, Serial/USB/LAN/Wi-Fi 직접 연결, HL7/GDT 등 연동을 공식 제공한다. 다만 API는 유료 서비스로 활성 구독·신청·승인·API Key·Webhook·IP 등록이 필요하다(미국 FAQ 기준). LB120은 CSV/이미지 export, LookinBody Web은 Excel export를 지원한다.

연동 단계별 PoC 적합도:

| 단계 | 방식 | 기술 난이도 | PoC 적합도 |
|---|---|---|---|
| 0 | 수동 입력 | 낮음 | 매우 높음 |
| 1 | PDF/이미지 업로드 | 낮음~중간 | 매우 높음 |
| 2 | CSV/Excel 내보내기 | 중간 | 높음 |
| 3 | 사용자 동의 기반 앱 데이터 공유 | 중간~높음 | 중간 |
| 4 | 공식 API/파트너 연동 | 높음 (유료·승인·계약) | 중간 (공모전 이후 본계약 단계) |

가장 현실적인 최소 방식은 **수련생 동의 기반 결과지 업로드**: ① 결과지 사진/PDF 업로드 → ② 핵심 지표(체중·골격근량·체지방률·부위별 균형·ECW Ratio·위상각)만 추출 → ③ 수업 기록(수업명·날짜·강사 피드백·감각 메모)과 연결 → ④ 월간 리포트 생성 → ⑤ 성과 후 Excel/CSV→LB120→Web API 순 고도화. 개인정보보호법상 수집 목적·항목·보유기간·동의 거부권 고지와 최소 수집 원칙을 준수한다.

### 4) 요가원 PoC는 장비 없이 시작하는 게 가장 빠르다

세 시나리오: A) 인바디 보유 프리미엄 스튜디오(설득력 최상이나 보유 업장 제한적), B) 외부 측정 결과 업로드형(비용 낮고 일반 요가원도 가능, 초기 PoC 최상), C) 인바디 없이 수업 기록만 먼저(가장 빠르고 브랜드 충돌 적음).

권장: **B+C를 기본으로 하고 A를 쇼케이스로 붙인다.**

| 단계 | 기간 | 내용 | KPI |
|---|---:|---|---|
| 1 | 2주 | 스튜디오 3~5곳에서 수업 언어·감각 기록만 운영 | 동의율, 기록 성공률, 강사 부담 |
| 2 | 4~6주 | 수련생 일부 외부 인바디 결과지 업로드 | 업로드율, 추출 성공률, 리포트 열람률 |
| 3 | 8~12주 | 인바디 보유 업장/협력 측정 공간에서 월 1회 측정 | 리포트 만족도, 재등록 의향, 수련 지속률 |
| 4 | 발표 | 샘플 리포트·운영 데이터 제시 | "인바디 수치 + 수업 맥락" 차별화 증명 |

### 5) 핵심 리스크와 대응

- **API 즉시 사용 불가 가능성** → 0~2단계 비API 방식으로 검증, API는 확장 단계.
- **장비가 비싸 직접 도입 어려움** → 기존 보유 업장·외부 업로드·270S/380 제한적 PoC·장비비 별도 과금.
- **수치 중심 평가와 요가 브랜드 충돌** → 체중·체지방률을 전면에 두지 않고 균형감·호흡·수련 흐름의 보조 지표로 전환.
- **수련생의 체성분 수치 부담** → 선택형 참여, 비공개 리포트, 수치 숨김 옵션.
- **녹음·건강 데이터 개인정보 리스크** → 명시적 동의, 최소 수집, 보관 고지, 삭제권, 비식별화.
- **의료기기·건강 표현 리스크** → "진단/치료/개선/처방" 대신 "참고/경향/수련 흐름/신체 인식".
- **강사 업무 부담** → 자동 요약, 강사 확인 1분, 감각 메모는 수련생 자가 입력.
- **인과관계 오류** → 상관 기록으로 표현, 식사·수면·타 운동 영향 함께 고지.
- **요가 환경 STT 정확도**(음악·호흡·산스크리트어·움직임 소음) → 마이크 위치·용어 사전·강사 검수 테스트, 음성 인식 성공률을 KPI에 포함.

## 몸소에의 함의

몸소는 인바디 장비 유통이 아니라 **맥락 데이터**로 차별화한다. 인바디가 "무엇이 변했는지"를 보여준다면, 몸소는 "어떤 수업 언어·피드백 속에서 그 변화가 기록되었는지"를 보여준다. 인바디 데이터는 몸의 정량 기록, 몸소 데이터는 수업의 정성 기록이 되고, 둘이 결합되면 재등록 상담·개인 리포트·강사 피드백 개선에 쓰이는 장기 웰니스 기록이 된다.

실행 우선순위:
1. PoC를 장비 없이(결과지 업로드·수동 입력) 먼저 돌려 "인바디 수치 + 수업 언어 + 감각 기록"의 가치를 검증한다.
2. 과금은 리포트 레이어를 본체로, 장비는 별도 옵션(Hardware Add-on)으로 분리 설계한다.
3. 리포트에서 체중·체지방률을 전면 배치하지 않고 평가가 아닌 "자기 이해의 참고값"으로 프레이밍한다.
4. API 연동은 확장 단계로 미루고, 초기엔 CSV/Excel·이미지 업로드로 검증한다.

이 포지션이 라이프스타일 코칭·건강 데이터 활용 방향과도 부합하며 가장 방어적이고 선명하다.
