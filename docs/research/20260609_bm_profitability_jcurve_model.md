# 몸소 BM·수익성 시나리오 모델 — 적자 저점이 얕고, ~20여 업장에서 BEP를 통과해 패스 GMV로 margin이 점프하는 J커브

> TL;DR: 개발비 0·스폰서 인프라로 PoC 적자 저점이 −100~250만원에 그치고, 2027 영업개시 구간 유료 ~20여 업장에서 손익분기를 통과한다. 비용은 BYOK/BYOC 믹스 상승으로 하방하고, 네트워크 임계 후 패스 GMV 수수료가 SaaS 매출 위에 얹히며 gross margin이 비선형으로 점프한다.

- 작성일: 2026-06-09
- 질문: 몸소의 BM에서 비용·수익을 시점·시나리오별 함수로 계산하면 J커브가 언제·어떻게·얼마만큼 그려지는가.

## 배경

확정 BM의 핵심은 두 층의 수익 구조다.

- **회원용(수련생) 앱 = 무료.** 사용자 네트워크 풀이 충분히 확보될 때까지 절대 과금하지 않고(Facebook·당근마켓·imweb 식), 임계 이후에야 패스권(몸소 패스) 장사가 가능 → 회원앱 수익은 네트워크 임계 전까지 0, 임계 후 GMV 수수료.
- **강사용 앱 = freemium:** 프리(무료) → 프로/맥스(유료) → 엔터프라이즈(전담 매니저·인바디 기기 연동·사업자 공제 지원).
- **시장 3단:** 1차(SOM) 전국 요가원 → 2차(SAM) 전국 웰니스 사업장 → 3차(TAM) 글로벌. PoC 4개월(2026.07~11) → 2027 본격 영업.
- **J커브 서사:** 비용곡선↓(BYOK/BYOC 전환 확산) + 수익곡선↑(B2B 구독 → 후행 플랫폼 패스 수수료) → gross margin 점프. 종착점 = 인바디 파트너십(데이터 제공 → 패스권 사업 → 웰니스 전 도메인 확장 → 엑싯/지속 선택).

**모델의 핵심 분기 — 인프라 부담 주체에 따라 플랜 가격이 2갈래.** AI(Anthropic Claude)·객체 스토리지(Naver Object Storage) 비용을 기본은 momso(회사)가 서버사이드로 부담하되, 원하는 고객은 자기 키·자기 저장소(BYOK/BYOC)로 쓰게 하고 그만큼 가격을 낮춘다.

| 인프라 버전 | 변동비 부담 주체 | 고객 체감 | 플랜 가격 |
|---|---|---|---|
| **momso-인프라판** | momso(서버사이드) | 비싸지만 편함(턴키) | 높음 (가격 = p) |
| **BYOK/BYOC판** | 고객(자기 키·저장소) | 싸지만 변동비 직접 부담 | 낮음 (가격 = p − Δ) |

`Δ` = momso가 변동비를 떠안지 않으므로 깎아주는 폭(고객 절감 변동비 + 마진 일부 이전). 네이버 CLOVA·Anthropic 제휴 시 서버사이드 단가도 인하 → 비용곡선 추가 하방. BYOC 연결 시 "양쪽에 낸다"는 이중 결제 인상이 안 들도록 패키지형 요금으로 복잡성을 숨긴다.

## 방법

확정 BM과 take rate·GMV 스팟체크를 입력값으로 받아, 시점·시나리오별 비용·수익을 함수로 모델링했다. 규율은 다음과 같다.

- 외부 수치는 '검증 앵커' 표의 값만 출처와 함께 사용.
- 미확정 값(티어 가격·전환율·BYOK 믹스·패스 침투율 등)은 파라미터(변수)로 두고 **보수/기준/공격** 예시값으로 시뮬레이션. 예시값은 추정이며 PoC 실측·확정 대상.
- 시간축 단위는 분기(Q)/연(Y), 화폐는 만원(별도 표기 시 억원). `N` = 활성 **유료** 업장 수(프리 플랜은 매출 0이므로 제외).

**비용 함수.** momso가 서버사이드로 부담할 때 업장당 월 변동비:

```
C_var(업장,월) = C_AI + C_STT + C_storage
              = (수업수 × 수업당_AI단가) + (수업수 × 수업당_STT단가) + 누적저장_단가
```

업장당 월 변동비 예시값 (Sonnet 기준, 월 80수업 가정):

| 항목 | 단가(앵커) | 월(80수업) | 출처 |
|---|---|---|---|
| AI (Claude Sonnet 4.6) | 수업당 ~190원(캐시 적용 ~100~150원); Opus 4.8 ≈ 2배 | **1.5만~3만원** | Anthropic 공식(Sonnet $3/$15, Opus $5/$25 per 1M, 캐시 0.1×, 배치 50%) |
| STT (CLOVA Speech) | 수업당 ~540원(보수 상단 — 전사 API 하위가 기준; CLOVA 정가는 분당 28원이라 PoC기 스폰서 제휴 대상; 온디바이스 Whisper로 0 근접) | **약 4.3만원(80수업)** | NCP 정가 기준(2026-06-12 갱신) |
| 저장 (Naver Object Storage) | 월 60분 압축음성 ~30~50MB, 업장당 월 3~4GB 누적 | **수백~수천원**(누적 증가) | NCP 공식(요금계산기로 확정) |
| **합계** | | **≈ 4만~8만원/업장·월** | |

`infra_bearer = 고객(BYOK/BYOC)`이면 `C_AI`·`C_storage`가 고객 부담으로 이전 → momso 변동비는 control plane 오케스트레이션 비용만 남아 점근 최소. 레버 파라미터: 모델 선택(Opus ≈2배), 캐시 적용률(0.1×까지), 배치 50% 할인, 스폰서 할인.

활성 업장 N개로 스케일할 때 BYOK/BYOC 채택 비율 `b`(0~1)를 반영하면:

```
변동비_총(월) = N × C_var × (1 − b)        ← b↑일수록 momso 부담↓ (비용곡선 하방)
```

**고정비·일회성.** PoC기 고정비(월) ≈ 10만~25만(도구 구독·서버 기본료). 인력 고정비는 PoC 추가 채용 없음(영업=유동환 네트워크, 개발=김성균) → 인건비 0 가정. 일회성 초기비 = 개발 0 + 법무 30만~80만 + 마이크/녹음 키트 20만~60만 + 도메인 등 ≈ 50만~150만원.

```
총비용(기간) = Σ[ N × C_var × (1−b) ] + 고정비 + 일회성(초기 1회)
```

**수익 함수.** 1단계는 B2B 구독(SaaS), 2단계는 패스 GMV 수수료(네트워크 임계 후).

```
B2B_연매출 = Σ_티어 ( 업장수_티어 × 티어월가_티어 × 12 )
플랫폼_연수익 = pass_on × N_active × (업장당 월 패스GMV) × take_rate × 12
이익(기간) = 총수익 − 총비용
```

## 발견

### 티어 가격 (원가·시장 기준 산출, 지불의사는 PoC 실측 후 최종)

| 티어 | 사용량 정의 | momso-인프라판 | BYOK/BYOC판(p−Δ) | 1년 약정(−20%) | 산출 근거 |
|---|---|---|---|---|---|
| 프리 | 월 녹음 10회·수련생 10명·보관 30일 | 0 | 0 | — | 업장당 월 ~0.6만 부담 = 획득비용(캡으로 통제) |
| 프로 (p1) | 월 녹음 45회·수련생 40명 | 9.9만 | 6.9만 | 7.9만 | 변동비 ~3.15만(45회×700원) → 마진율 68.2%; 바디코디 최저가(선납 13.86만)보다 낮은 진입 쐐기 |
| 맥스 (p2) | 무제한(공정사용 월 120회)·수련생 무제한·스토리지 확장 | 19.8만 | 14.8만 | 15.84만 | 변동비 5.6–8.4만(80–120회×700원) → 마진율 57.6–71.7%(평균 80회 71.7% / 헤비 120회 57.6%); 약정가 = 바디코디 약정과 동일("같은 가격에 AI 포함") |
| 엔터프라이즈 (p3) | 다지점·전담 매니저·인바디 연동·공제 지원 | 39.8만부터(견적) | — | 견적 | 전담 매니저 인건비(계정당 ~20만, 1인 20계정 가정) 포함이라 고정가 아닌 견적 |

- Δ(BYOK/BYOC 할인폭) = 이전 변동비 + 마진 일부: 프로 Δ3만(이전 원가 ~3.15만), 맥스 Δ5만(이전 원가 ~5.6만) — 정직한 원가 이전.
- **원가 기준 갱신(2026-06-12):** 위 변동비·마진은 NCP 정가 확정 신기준(수업당 변동비 700원 · 프로 GM 68.2% · 맥스 71.7%/57.6%)으로 교체됨. 6/9 작성 시점의 600원/GM 73%는 폐기.
- ARPA 정합: 프로:맥스 5:5·BYOK 30% 시 평균 월 ~13.7만(연 164만), 약정 반영 시 ~130만대 → ARPA 범위 120–180만(기준 150만)에 부합.

### 패스 GMV 입력값

- **업장당 월 패스 GMV** = 수련생 20명 × 회당 2.5만(오붓 패스 회당 앵커) × 월 4회 = **200만/업장**(보수 앵커). 바디코디 역산(업장당 월 3,300만)은 헬스장·PT 포함 평균이라 상한선으로만.
- **take_rate(단계화):** 보수 5~8% / 기준 10~15% / 공격 20~25%. ⚠️ 한국 단일 중개수수료 10% 돌파 = 규제·여론 리스크(배민) → 결제중개(3~4%)와 마켓플레이스 수수료(10~20%) 분리 표기 권장.

### 시점 × 시나리오 시뮬레이션

시점 3구간: PoC(2026H2) · 2027 영업개시 · 확산기(2028+, 패스 on). B2B는 유료 ARPA(업장당 연 B2B 매출) 단일 변수로 묶음.

| 파라미터 | 기호 | PoC(2026H2) | 2027 영업개시 | 확산기(2028+) |
|---|---|---|---|---|
| 활성 **유료** 업장수 | N | 1 (빅블루, 사실상 무료 PoC) | 보수 50 / 기준 100 / 공격 100 | 보수 500 / 기준 1,000 / 공격 2,500 |
| 유료 ARPA(연, 만원) | A | 0 (PoC 무료) | 120~180 | 160~200 |
| BYOK/BYOC 믹스 | b | 0% | 10~20% | 보수 30% / 기준 50% / 공격 70% |
| 업장당 월 변동비(만원) | C_var | 4~8 | 4~8 | 4~8 (단가 인하 시↓) |
| 패스 스위치 | pass_on | 0 | 0 | **1** (임계 후) |
| 업장당 월 패스 GMV(만원) | G | — | — | 200 |
| take rate | t | — | — | 보수 7% / 기준 12% / 공격 20% |

기준 시나리오 시점별 계산:

- **(a) PoC 2026H2 (4개월):** 유료매출 0. 비용 = 일회성 50~150만 + 월 10~25만 × 2~3개월 ≈ 100만~250만원. 이익 ≈ **−100만~−250만원**(J커브 적자 저점). 모두의창업 활동자금 2백만 + InBodyLIKE PoC 지원 + 빅블루 매출로 외부투자 없이 완주 가능.
- **(b) 2027 영업개시 (N=100, A=150만, b=15%, C_var=6만):** B2B 매출 = **1.5억/년**. 변동비 = 100×6만×12×0.85 ≈ **6,120만(≈0.61억)**. 고정비 ≈ 0.2억. 이익 ≈ 1.5 − 0.61 − 0.2 ≈ **+0.7억/년 → 흑자 전환**. BEP 업장수 ≈ 고정비·일회성 ~2,000만 ÷ 업장당 연 기여이익(150 − 61.2 = 88.8만) ≈ **약 23업장**.
- **(c) 확산기 2028+ (N=1,000, A=180만, b=50%, C_var=6만, 패스 on, t=12%, G=200만):** B2B 매출 = **18억/년**. 변동비 = 1,000×6만×12×0.5 = **3,600만(≈0.36억)** — N은 10배인데 BYOK 50%라 변동비는 (b)의 6배 수준에 그침. 플랫폼 수익 = 1,000×200만×12%×12 = **28.8억/년**(활성 업장 전부 패스 가정 상한). 고정비 ≈ 2억. 이익 ≈ (18+28.8) − 0.36 − 2 ≈ **+44.4억/년**.

## 몸소에의 함의

핵심 레버가 J커브에 주는 영향(민감도):

| 레버 | 영향 | 방향 |
|---|---|---|
| **① take rate (t)** | 확산기 이익의 최대 변동 요인. 7%→20%면 패스 수익 8.4억→24억(업장 500 기준)으로 ~3배. 단 10% 돌파는 규제·여론 리스크 → 상방에 천장. | 흑자 후 기울기 직접 결정 |
| **② 활성 유료 업장수 (N)** | B2B·플랫폼 양쪽의 곱셈 인자. J커브의 x축 진행 속도이자 절대 규모. 영업 실행력에 가장 의존. | 곡선 전체 스케일 |
| **③ BYOK/BYOC 믹스 (b)** | 비용곡선의 하방 기울기. b가 0→50→70%로 오르면 N이 10배여도 변동비가 5~6배에 그쳐 margin 개선. | BEP 시점 앞당김·margin↑ |
| **④ 패스 침투율 (pass_pen)** | '활성 업장 전부 패스' 가정을 현실화. 침투율 30%면 확산기 패스 수익이 1/3로. 가장 미검증·낙관 위험 큰 가정. | 확산기 상방 보정 |
| **⑤ 유료 ARPA / 티어가격·전환율** | 1단계 적자 저점 깊이와 BEP 업장수를 결정. ARPA 120→180만이면 BEP 업장수 ~23→~17로 감소. | 적자 저점·BEP |

요약하면, momso의 J커브는 ① 개발비 0·스폰서 인프라로 적자 저점이 얕고(PoC −100~250만), ② 유료 ~20여 업장에서 BEP를 통과하며, ③ BYOK 믹스 상승으로 비용곡선이 하방하고 ④ 네트워크 임계 후 패스 GMV 수수료가 SaaS 위에 얹히며 margin이 점프하는 구조다.
