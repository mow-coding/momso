# 몸소 레퍼런스 체크(출시 시장) — 6요소를 통째로 구현한 출시 경쟁자는 없다

> TL;DR: 출시 시장은 ⓐ요가 수업 빌더, ⓑ스튜디오 예약/CRM, ⓒ인바디 체성분 생태계로 갈라져 있고 어느 쪽도 나머지 축을 건드리지 않는다.
> 몸소의 6요소(음성 맥락 캡처·인바디 세분화 데이터·강사 HITL 검수 공유·데이터 주권·요가 도메인·예약+결제 BM)를 통째로 구현한 출시 제품은 없다.
> 인바디 데이터 '접근'은 상품화된 통합이라 해자가 아니며, 진짜 방어선은 요가 도메인 지식·맥락 캡처·HITL·데이터 주권의 결합이다.

- 작성일: 2026-06-08
- 질문: 기능 정의 6요소(①수업 음성 맥락 캡처 ②인바디 좌우/세분화 체성분 ③강사 HITL 검수 공유 ④데이터 주권/Web2.5 ⑤요가 도메인 AI 래퍼 ⑥지도 예약+결제 BM) 기준으로, 같은 조합을 구현한 경쟁/유사 서비스가 출시 시장에 존재하는가.

## 배경

자문 피드백을 반영해, 몸소 기능 정의의 6요소를 기준으로 경쟁·유사 서비스를 점검했다. 단 이 1차 패스는 **"출시 제품"만 검증**한다. 미출시(스텔스 스타트업·AC/VC 포트폴리오)와 특허(FTO·선행사례)는 미검증 영역으로 남기며 2차 패스가 필요하다(문서 끝 참조).

## 방법

5개 각도로 팬아웃해 다수의 공개 소스를 수집한 뒤, 추출한 주장들을 적대적으로 교차검증(confirmed/killed로 분류)했다. 본 보고서에는 검증을 통과한 주장과, 인용 시 표기 단서가 필요한 항목만 남긴다.

### "몸소 = 고유한 조합" 근거 (출시 시장 한정, confidence: high)

어떤 단일 제품도 **음성 맥락 캡처 + 인바디 세분화 데이터 + 강사 HITL 검수 공유 + 데이터 주권**을 동시에 갖지 않는다. 빌더는 음성·BIA가 없고, 예약/CRM은 콘텐츠 캡처가 없고, 인바디 생태계는 도메인·맥락·주권·HITL이 없다.

### 가장 가까운 경쟁자 Top 3 + 방어 논리/리스크

1. **InBody LookinBody Web (1st-party)** — 체성분+공유 축을 인바디가 직접 점유.
   - 방어: 자동 동기화/공유(강사 검수 프레이밍 ✕), 요가 도메인·음성맥락·사용자소유 저장 ✕.
   - 리스크: 인바디가 도메인 확장 시 **가장 직접적 위협**이자 동시에 **잠재 파트너**(인바디 협업 BM과 직결).
2. **Tummee** — 출시·대규모 요가 강사 플랫폼, 카테고리 ⓐ 직접 경쟁.
   - 방어: 공유물이 *사전 포즈 시퀀스 시각자료*일 뿐, 수업 후 음성맥락+인바디 융합 HITL 리포트가 아님. 음성=재생 cue뿐. 인바디 전무.
3. **Terra/Technogym 류 인바디 통합 미들웨어** — #2를 벤더-투-벤더 통합으로 점유.
   - 방어: 데이터 파이프일 뿐 요가 코칭앱 아님. 음성/수업맥락·트레이너→수련생 리포트 레이어 없음.

## 몸소에의 함의

- **인바디 데이터 "접근" 자체는 해자가 아니다.** 서드파티가 짧은 시간이면 붙이는 **상품화된(off-the-shelf) 통합**이다(단, 디바이스 소유+인바디 승인 게이트는 있음). → "인바디랑 연동한다"를 차별점으로 내세우면 약하다.
- **단, 좌우 밸런스·근육 비대칭·세분화(segmental) 데이터는 off-the-shelf 미들웨어가 노출하지 않는다**(Terra는 체지방·BMI·BMR 등 전신 집계값만). 세분화 데이터는 인바디 원천/직접 연동(Technogym식)에만 있음 → 약하지만 방어 논리로 사용 가능.
- **진짜 해자 = 요가 도메인 UX + 수업 맥락 캡처 + 강사 HITL 검수 + 데이터 주권의 결합.** "데이터 기술/자격증"이 아니라 "비정형 요가 도메인에서 의미 있는 기록을 추출하는 도메인 지식"으로 차별성을 프레이밍.

**피치덱 '레퍼런스 체크' 슬라이드용 요약(3~5문장):**

> 몸소는 ①수업 음성의 맥락 보존 캡처 ②인바디 좌우/세분화 체성분 데이터 ③강사 HITL 검수 후 수련생 공유 ④사용자 데이터 주권(Web2.5)을 하나의 요가 도메인 제품으로 결합한다. 시장 점검 결과 이 조합 전체를 구현한 출시 경쟁자는 확인되지 않았다 — 요가 수업 빌더(Tummee, The Class Plan)는 음성·인바디가 없고, 스튜디오 예약/CRM(StudioMate, Mindbody)은 수업 콘텐츠 캡처가 없으며, 인바디 자체 생태계(LookinBody Web, Terra, Technogym)는 체성분 데이터에 머물 뿐 요가 맥락·강사 검수·데이터 주권이 없다. 인바디 데이터 접근 자체는 짧은 시간이면 붙는 상품화된 통합이라 누구의 해자도 아니며, 좌우 비대칭/세분화 데이터는 off-the-shelf로 얻을 수 없다. 몸소의 방어선은 데이터 기술이나 자격증이 아니라, 비정형 요가 도메인에서 의미 있는 기록을 추출하는 도메인 지식과 그 위의 UX다.

## 한계 · 다음 질문

**범위 한계:** 이 1차 패스는 "출시 제품"만 검증했다. 아래는 미검증 영역으로, 2차 패스가 필요하다.

1. **특허(FTO·선행사례)**: 체성분+음성/수업맥락 결합 코칭, BIA 기반 개인화 운동처방, 요가 수업 기록·공유 시스템 → freedom-to-operate 리스크/선행사례 여부.
2. **미출시·스텔스 스타트업**: 국내 AC/VC(프라이머·본엔젤스·스파크랩·블루포인트·퓨처플레이) 포트폴리오, CES Innovation Awards·데모데이 웰니스/무브먼트 데이터 스타트업(예: CES2025 체성분 분석 트레이닝 머신 등 신호 포착됨).
3. **인바디 로드맵**: 인바디가 LookinBody/생태계를 요가·강사 HITL 코칭으로 확장할 계획 시그널.
4. **국내 직접 대체 타깃**: 오붓·그로우·자버·bsweat가 음성 수업 기록·체성분 연동을 로드맵에 두는지(1차에선 StudioMate/Mindbody만 다룸).

**인용 시 주의(검증 노트):**

- Tummee 규모 수치(1M+ 시퀀스 등)는 **벤더 자기보고**, 제3자 미감사 → "self-reported"로 표기.
- Technogym–InBody 통합은 **2017년 발표**(현행 커넥터로 현재성 유지) → "데이터 레이어 한정, 2017 시점" 명시.
- Terra "커모디티화"는 인바디 승인 게이트 있음 → "디바이스 소유+승인 게이트 있는 off-the-shelf 통합"으로 정확화.
- **사용 금지(반증됨)**: "Trainerize/FitSW가 인바디·음성 없음" 류 주장은 검증에서 반증 → 슬라이드에 쓰지 말 것.
