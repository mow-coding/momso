# 요가 도메인 고민과 몸소 제품 전략 — 몸소는 "녹음·요약 앱"이 아니라 동의·검수·노출 통제·장기 아카이빙 서비스다

> TL;DR: 몸소의 본질은 수업을 녹음해 요약해주는 것이 아니라, 사라지는 지도자의 언어와 수련생의 감각을 지도자가 검수하고 공유 범위를 통제한 뒤 회원별 장기 수련 기록으로 남기는 것이다. 요가는 개인정보와 지도자 노하우가 가장 민감한 도메인이므로 MVP는 대형 그룹이 아니라 1:1·소그룹·프리미엄 케어형 수업에서 시작한다. Tiro는 세션 단위 전사 파트너, InBody는 첫 정량 데이터 파트너일 뿐 몸소의 중심이 아니다.

- 작성일: 2026-05-29
- 질문: 요가 현장의 고민(개인정보·저작권·녹음 장비·InBody·CRM)을 어떤 제품 원칙으로 바꿔야 몸소가 단순 녹음·요약 앱이 아닌 책임형 수련 기록 서비스가 되는가?

## 배경

요가원을 운영하고 수련생을 지도해본 사람만 감지하는 불편함·위험·가능성을, 몸소 제품 설계 원칙으로 재구성한 문서다. 출발점은 유동환이 정리한 요가 현장의 다섯 가지 고민(개인정보, 지도자 노하우·저작권, 녹음 기기·서버, InBody 협업, 메모 공유·CRM)이다.

핵심 전제 구분은 다음과 같다.

- 나쁜 정의: "수업을 녹음하고 AI가 요약해서 회원에게 보내준다."
- 좋은 정의: "수업 중 사라지는 지도자의 언어와 수련생의 감각을, 지도자가 검수하고 공유 범위를 통제한 뒤, 회원별 장기 수련 기록으로 남긴다."

## 방법

요가 현장 운영 경험에서 나온 정성적 우려를 제품 리스크·설계 원칙·MVP 범위로 번역하고, 용량·비용은 빅블루 요가의 실제 수업 운영 수치를 기반으로 단순 추정했다. InBody·CRM 연동은 PoC에서 검증할 질문 형태로 정리했다.

## 발견

### 1. 개인정보 — 정해진 항목이 아니라 "상황 속 민감 대화"

기존 예약·회원관리 SaaS가 다루는 것은 이름·연락처·수강권·예약·결제처럼 정해진 항목이다. 반면 몸소가 다루는 것은 수업 중 나온 몸 상태, 통증·긴장·불편감, 감정 표현, 지도자의 개별 피드백, 타인에게 한 조언, 지도자의 수업 노하우, 자연스럽게 섞인 사적 대화다. 즉 단순 개인정보가 아니라 상황 속에서 생성되는 민감 대화 정보다.

요가는 PT·필라테스보다 홍보가 보수적이다. Before & After·체형 변화 홍보가 흔한 PT/필라테스와 달리, 요가원은 수업 중 촬영 금지가 많고, 몸·감정 노출을 조심하며, 공간을 내밀한 소통의 장소로 여긴다. 따라서 동의서 한 장으로 개인정보 문제를 해결할 수 있다고 보면 안 된다.

그룹 수업의 본질적 위험: A에게 가는 리포트에 B·C·D의 정보(B의 허리 통증, C의 자세 교정 피드백, D의 감정 상태)가 섞일 수 있다. AI 필터링은 100% 보장이 안 되고, 사고 한 번에 요가원 신뢰가 무너진다. 이것이 몸소의 가장 큰 리스크다.

설계 원칙:

| 원칙 | 설명 |
|---|---|
| 수업별 명시 동의 | 가입 시 한 번이 아니라 녹음되는 수업마다 고지 |
| 원본 비공개 | 음성 원본·전체 전사본은 회원에게 기본 비공개 |
| 지도자 검수 필수 | AI 결과를 바로 보내지 않고 강사 확인 후 공유 |
| 회원별 리포트 분리 | A 리포트에는 A 해당 내용만 포함 |
| 민감정보 제거 | 이름·타인 피드백·개인 통증·사적 발언 자동 제거 후보 |
| 녹음 일시정지 | 개인 상담·민감 대화 시 녹음 끄거나 비공개 메모 전환 |
| 공유 범위 표시 | 공유 가능 / 내부 기록 / 삭제 / 개인 리포트 제외 선택 |
| 접근 로그 | 누가 어떤 기록을 열람했는지 기록 |

MVP 권장: 빅블루 요가 내부에서 시작 → 1:1 우선 → 2~4인 소그룹 선택 적용 → 녹음 동의 회원만 → 원본 공유 금지 → 지도자 검수 후 개인 리포트만 공유 → 그룹 수업은 PoC 후반부 제한 테스트. 한 문장: "AI가 알아서 지워준다"가 아니라 "처음부터 위험한 공유가 일어나지 않는 구조를 설계한다."

### 2. 지도자 노하우·저작권 — 노출 통제가 핵심 기능

요가 시장에서 지도자의 시퀀스·큐잉·가이드·수업 구성은 경쟁력이다(TTC 유인물 외부 공유 제한도 같은 이유). 원장·강사는 시퀀스 유출, 큐잉 복제, 자료 재판매, 노하우가 회원 리포트로 과도하게 남는 것, LLM으로 지식이 재가공되는 것을 우려한다. 몸소는 회원 개인정보뿐 아니라 지도자 지식재산도 보호해야 한다.

요가원이 얻는 것(케어 품질·프리미엄 이미지·장기 기록·재등록 유도·전문성 전달·상담 누락 감소)이 잃을 수 있다고 느끼는 것(노하우 노출·수업 방식 복제·민감 대화 유출·검수 업무 증가·기존 CRM 병행 부담)보다 커야 한다. 그래서 "노출 통제"를 제품 중심 기능으로 넣어야 한다.

노출률 설정(스킬 다분화) 모드:

| 모드 | 설명 | 공유 범위 |
|---|---|---|
| 회원 몸 상태 중심 | 회원 감각·몸 상태·과제 중심 | 회원 공유 가능 |
| 수업 피드백 중심 | 주요 큐잉·개인 피드백 균형 | 지도자 검수 후 공유 |
| 지도자 노하우 보호 | 시퀀스·전문 큐잉·구조를 내부 기록만 | 외부 공유 금지 |
| 마케팅 공개 모드 | 후기·한 문장 피드백·카드뉴스 문장 | 공개 전 검수 |
| TTC/교육 보호 모드 | 교육 자료·시퀀스·유인물 자동 비공개 후보 | 내부 전용 |

다운로드 정책: 회원은 원본 음성·전체 전사본 다운로드 불가, 검수 리포트만 열람. 공유 이미지 워터마크 검토, export 권한별 제한, 외부 공유 링크 만료, 원장이 다운로드 허용 여부 직접 설정. 약관보다 제품 구조가 먼저다 — 위험한 행위를 실제로 어렵게 만들어야 한다.

### 3. 녹음 기기·서버 — 용량·비용 구조

장비별 트레이드오프: 핸드폰(배터리·용량·알림 방해·장시간 안정성), 노트북(10평 정도 가능하나 큰 공간 음질 저하·이동성 낮음), 블루투스 핀마이크(큰 공간·작은 목소리·배경음악에 유리하나 충전·연결·착용 부담).

용량·시간 추정(빅블루 기준):

- 70~80분 수업 1회: 약 80MB
- 월 수업: 그룹 약 40타임 + 개인 약 20타임 = 약 60타임
- 월 녹음량 약 5GB, 월 녹음 시간 약 5,000분

| 규모 | 월 수업 수 | 월 녹음 시간 | 월 저장 용량 |
|---|---:|---:|---:|
| 1개 업장(빅블루) | 60타임 | 약 5,000분 | 약 5GB |
| 10개 업장 | 600타임 | 약 50,000분 | 약 50GB |
| 20개 업장 | 1,200타임 | 약 100,000분 | 약 100GB |

단, 실제 API 비용은 저장 용량보다 처리 분수·실시간 여부·요약 횟수·재처리 횟수에 더 크게 좌우될 수 있다.

MVP 장비 전략은 세 세팅만 검증: 노트북/태블릿 룸마이크(작은 공간 기본값), 스마트폰 단독(개인·소규모, 모바일 MVP 가능성), 블루투스 핀마이크(큰 공간·음악 수업 품질 옵션). 초기 가이드: 10평 내외 조용한 공간은 룸마이크, 더 크거나 음악 있는 수업은 핀마이크.

Tiro 요금제 주의: 소비자용 무제한 요금제와, 몸소 앱이 API로 대량 호출할 때의 비용은 다를 수 있다. Tiro에 확인할 것 — 과금 단위(분당/요청당), 실시간 vs 파일 업로드 비용 차이, 요약·템플릿 별도 과금 여부, 월 5만~10만 분 처리 시 할인 구간, PoC 크레딧, 음성 원본 저장 비용 포함 여부.

### 4. InBody 협업 — 첫 정량 데이터 파트너이지 본질은 아니다

InBody는 체수분·단백질·무기질·체지방 등 신체 변화를 수치로 보여줘 헬스장·PT샵에서 강력하다. 그러나 요가에서는 의문이 생긴다 — 요가 핵심 목적이 체지방 감소인가, 명상·마음 변화를 인바디로 설명할 수 있나, 자세 교정은 체성분보다 카메라 기반 체형 분석이 직관적이지 않나, 수련자가 측정 밴드를 착용하려 할까, 질적 경험을 숫자로 환원하면 오히려 요가답지 않아 보이지 않나.

몸소의 본질은 수업 언어·지도자 피드백·수련생 감각·장기 아카이빙·몸의 변화와 수업 맥락의 연결이다. InBody의 역할은 첫 정량 신체 데이터, 건강관리 참고 지표, PoC 외부 파트너 실험 데이터, 자기관리 관심층 설득 자료다.

- 나쁜 표현: "몸소는 인바디 데이터를 기반으로 요가 효과를 증명합니다."
- 좋은 표현: "몸소는 인바디와 같은 정량 신체 데이터를 수업 중 오간 언어·감각 기록에 연결해, 회원이 수련 과정을 더 입체적으로 이해하도록 돕습니다."

InBody가 유효한 지점: 건강관리·다이어트·근육량 관심 회원, 장기 수련 전후 변화, 자기관리 루틴, 신체 변화 보조 근거, 체형·통증 개선과 체성분 변화를 함께 보는 보조 자료. 다만 명상·심리 안정·자세 교정의 핵심 지표로는 한계. PoC 권장: InBody는 반드시 써야 하는 핵심축이 아니라 몸소가 연결할 수 있는 첫 번째 정량 데이터 파트너로 둔다.

### 5. 메모 공유·CRM — 지금 CRM을 만들면 안 된다

요가원은 이미 바디코디·스튜디오메이트 등 CRM으로 예약·스케줄·회원·수강권·결제·출석을 관리한다. 몸소가 리포트를 자동 공유하려면 수업 참석자를 알아야 하는데, 연동 없이는 수기 입력 부담이 크다.

처음부터 예약·결제·수강권 CRM까지 만들면 위험하다 — 바디코디 같은 기존 업체와 정면충돌, 개발 범위 비대, 차별점 희석, 핵심 기능(수업 기록·리포트) 검증 지연, 원장에게 또 하나의 복잡한 도구로 보임. 따라서 MVP는 CRM이 아니라 수업 기록·리포트 레이어여야 한다.

단계: ①수동 입력(수업명·일시·참여 회원·개인/그룹·동의 여부) → ②CSV/복붙(바디코디 예약 명단 붙여넣기, 회원명·전화 일부 매칭, 출석자 선택) → ③특정 CRM 연동(바디코디 API 가능 여부 확인, 스튜디오메이트 등은 후순위) → ④몸소 자체 예약 기능(장기, 지금 핵심 아님).

팀스페이스 재정의: 노션·Tiro·Clova Note의 팀스페이스는 같은 목적의 팀원이 공유하는 구조지만, 요가 수업 참여자는 같은 조직 팀원이 아니라 각자 다른 목적·사적 경험을 가진 개인이다.

| 구분 | 접근 권한 |
|---|---|
| 요가원 관리자 | 업장 전체 기록 관리 |
| 강사 | 본인 수업·배정 회원 기록 |
| 회원 A | 본인에게 공유된 리포트만 |
| 회원 B | 본인에게 공유된 리포트만 |
| 그룹 수업 참여자 | 서로의 리포트 접근 불가 |

핵심: 몸소는 공동 워크스페이스가 아니라, 업장 관리자가 회원별 접근권을 분리해 운영하는 수련 기록 공간이다.

### 6. 첫 타깃 — 대형 그룹이 아니라 1:1·소그룹·프리미엄 케어

적합한 첫 타깃: 1:1 개인 수련 요가원, 2~4인 소그룹, 프리미엄 케어형 스튜디오, 개인별 피드백이 중요한 필라테스/PT, 회원·지도자 관계가 깊고 재등록·장기 관리가 중요한 업장. 부적합: 10명 이상 대형 그룹만 운영, 일일권·체험권 비중 높음, 회전율 높고 개인 기록 가치 낮음, 녹음·동의·검수를 번거롭게 느끼는 센터.

연희동 전략은 네트워크·현장 검증·지역 협력 경험·도메인 신뢰라는 자산이 있지만, 그룹 수업 비중이 높고 일일권 회원을 팀스페이스에 넣기 번거로운 한계가 있다. 따라서 연희동은 첫 실험장·신뢰 자산으로 두되, 첫 유료 타깃은 연희동 안팎의 1:1·소그룹·프리미엄 케어형 업장이다.

확장 포지션: 요가는 작은 시장이 아니라 가장 까다로운 첫 시장이다. 요가에서 동의·검수·노출 통제·감각 기록을 풀면 필라테스·PT·재활운동으로 확장할 수 있다. InBody로 PT·헬스장이 쉬워 보여도 그 방향으로만 가면 정체성이 흐려진다.

### 7. 본질적 기능 — Voice Auto Archiving

핵심은 개별 녹음본 요약을 넘어선다: 수업 언어·감각을 기록하고, 수업 후에도 남기고, 회원별 수련 기록으로 변환하고, 인바디 같은 몸 수치를 수업 경험과 연결하고, 여러 회차를 장기 축적해, 다음 수업·피드백에 다시 활용한다.

| 구분 | Tiro | 몸소 |
|---|---|---|
| 기본 단위 | 한 회의/한 녹음 | 한 회원의 장기 수련 여정 |
| 핵심 기능 | 전사·요약·문서화 | 수업 기록·지도자 검수·개인 리포트·장기 아카이브 |
| 사용자 | 회의/업무 사용자 | 요가원·강사·수련생 |
| 공유 방식 | 팀/개인 노트 공유 | 회원별 접근권 분리 |
| 도메인 지식 | 범용 대화 | 요가/웰니스/몸 데이터 |
| 핵심 자산 | 음성 처리 품질 | 장기 기록 구조·도메인 리포트 설계 |

데이터 구조:

```text
회원
-> 수업 회차
   -> 수업 유형 / 참여자 / 녹음 동의
   -> 음성 원본 / 전사본 / AI 요약 후보
   -> 지도자 검수 메모 / 민감정보 제거 로그
   -> 개인 수련 리포트 / 인바디·체형·감각 데이터
-> 장기 수련 아카이브
-> 다음 수업 참고 요약
```

이 구조가 있어야 아카이빙이 된다. 매번 녹음 파일을 올리고 요약을 받는 것은 몸소가 아니다.

핵심 AI 스킬: ①요가 수업 구조 이해(호흡·워밍업·메인 시퀀스·쿨다운) ②큐잉 이해(정렬·호흡·감각 유도·관절·중심) ③회원 상태 분류(통증·긴장·가동범위·피로·감정) ④공유 위험 판단(타인 정보·개인 상담·노하우·사적 대화) ⑤리포트 변환(오늘의 요약·피드백·다음 과제·몸 상태 변화·장기 패턴) ⑥인바디 연결(체성분을 무리하게 요가 효과로 단정하지 않기).

Tiro가 안 될 경우 대안: OpenAI API로 전사·요약 파이프라인, 스마트폰·노트북 녹음 후 업로드, Whisper 계열 전사 검토, Tiro는 일반 유료 플랜으로 실험만, 초기에는 수동 업로드 + AI 요약 + 지도자 검수로 MVP 검증. 중요: 투자받아야 하는 이유를 "Tiro가 필요해서"로 만들면 약하다 — "요가·웰니스 수업 기록의 동의·검수·장기 아카이빙 구조를 만들기 위해서"라고 말해야 한다.

## 몸소에의 함의

**지금 해야 할 것**: ①빅블루 내부 1:1·소그룹 녹음 실험 ②수업별 동의 문구 ③지도자 검수 리포트 템플릿 ④민감정보 제거 체크리스트 ⑤원본·전사본·리포트 접근권 정책 ⑥월 녹음 시간·저장 용량 계산 ⑦Tiro API vs OpenAI API 비용 비교 ⑧FGI로 수련생이 원하는 리포트 확인.

**지금 하지 말아야 할 것**: 처음부터 전 요가원 대상 B2B SaaS, 대형 그룹 전체 자동 리포트 발송, 전체 전사본 다운로드 허용, 예약·결제 CRM 동시 구축, 인바디 수치를 요가 효과 증명처럼 말하기, Tiro 없으면 제품을 못 만든다고 전제하기, 소비자 앱에 과도한 SNS 기능.

**1차 MVP 정의**: 1:1 또는 소그룹 요가 수업에서 동의받은 음성을 기록하고, AI가 전사·요약 후보를 만들고, 지도자가 검수해 회원별 개인 수련 리포트를 공유하며, 이 기록을 장기 아카이브에 쌓는 서비스.
- 포함: 수업 등록 / 참여자 등록 / 녹음 동의 체크 / 파일 업로드 / 전사·요약 / 지도자 검수 / 개인 리포트 생성 / 회원별 공유 / 기록 누적.
- 제외: 전체 CRM / 결제 / 수강권 / 대형 그룹 자동 처리 / 인바디 자동 API 연동 / 소비자 SNS 전체 기능.

**발표자료에 바로 쓸 문장**:
> 몸소는 요가 수업 녹음본을 그대로 공유하는 서비스가 아닙니다. 수업 중 오간 지도자의 언어·피드백·수련생의 감각을 AI가 구조화하고, 지도자가 검수한 뒤, 회원별 개인 수련 리포트로 안전하게 축적하는 서비스입니다.

> 요가 수업에는 지도자의 노하우와 회원의 사적인 몸 상태가 함께 담길 수 있어, 몸소는 원본 음성·전체 전사본을 기본 비공개로 두고 동의·검수·노출 통제 구조를 제품의 핵심으로 설계합니다.
