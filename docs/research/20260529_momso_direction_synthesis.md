# 몸소 방향성 총정리 — 몸소는 '녹음·요약 앱'이 아니라 '책임 있는 수업 기록화' 서비스다

> TL;DR: 몸소의 1단계는 CRM이 아니라 수업 기록/공유 경험이며, MVP는 대형 그룹이 아니라 1:1·소그룹부터 시작한다. 핵심 자산은 전사 엔진이 아니라 `동의 → 전사 → 필터링 → 지도자 검수 → 개인 리포트 → 장기 아카이브` 구조다. Tiro는 전사/요약 기술 후보, InBody는 첫 정량 신체 데이터 파트너로만 위치시킨다.

- 작성일: 2026-05-29
- 질문: 일련의 팀 회의·도메인 고민·시장 리서치, 그리고 Tiro·InBody 협업 검토를 거친 뒤, 몸소가 무엇이고 무엇이 아닌지를 다시 고정하고 향후 2~4주의 검증 범위를 좁힐 수 있는가.

## 배경

몸소를 둘러싼 논의가 여러 갈래로 확장되면서, 외부 발표(InBodyLIKE)와 파트너 협의(Tiro)에서 서로 다른 말이 나올 위험이 생겼다. 이 보고서는 (1) 몸소의 정의를 팀 내부에서 다시 고정하고, (2) 외부 커뮤니케이션의 기준선을 만들고, (3) 다음 2~4주 동안 무엇을 만들고 검증할지 좁히기 위해 작성됐다.

핵심 인식: 요가·필라테스·PT처럼 몸을 다루는 수업에는 '기록 계층'이 빠져 있다. 수업 중 좋은 피드백과 감각은 대부분 기억에 의존해 사라진다. 이는 지도자의 게으름이 아니라, 몸의 체득에 집중하는 수업 본분에서 생기는 구조적 공백이다.

## 방법

- 일주일간의 팀 내부 회의(점심 회의·저녁 통화)와 유동환의 도메인 고민, 김성균의 거시 포지셔닝 리서치를 종합.
- 외부 LLM(GPT-5.5)을 활용한 발표자료 표현 리스크 스팟체크.
- Tiro 온보딩 및 B2B 미팅에서의 협력 조건 청취.
- 경쟁/인접 서비스(바디코디, 오붓/ClassPass, Tiro, InBody, 포인티 등)와의 차이 정리.
- 빅블루 요가원 실측치를 기준으로 한 사용량·저장량·비용 가정 산출.

## 발견

### 1. 몸소의 정의

몸소는 '요가 수업 녹음/요약 앱'이 아니다. 몸을 다루는 수업에서 사라지는 지도자의 언어·교정 피드백·수련생의 감각·정량 신체 데이터를, 동의·필터링·지도자 검수·노출 통제 구조 안에서 회원별 장기 수련 기록으로 전환하는 서비스다.

> 인바디가 몸의 스펙을 기록한다면, 몸소는 수업의 맥락을 기록한다.

### 2. 확정 방향 7가지

1. 1단계는 CRM이 아니다 — 예약·결제·출석·수강권이 아니라 수업 기록/공유 경험.
2. MVP는 대형 그룹이 아니라 1:1·2~4인 소그룹·프리미엄 케어형 수업부터.
3. 핵심 자산은 전사 엔진이 아니라 `동의 → 전사 → 필터링 → 지도자 검수 → 개인 리포트 → 장기 아카이브` 구조.
4. Tiro는 강력한 음성 처리 후보지만 종속되면 안 된다.
5. InBody는 첫 정량 신체 데이터 파트너이지 본체가 아니다.
6. 메시지는 'AI가 자동으로 보내준다'가 아니라 '지도자가 검수해 안전하게 남긴다'.
7. 데이터 주권·원본 보관·삭제권·학습 미사용은 부가 기능이 아니라 핵심 설계 항목.

### 3. 포지셔닝 — 피해야 할/잡아야 할 자리

피해야 할 포지션: 요가원 CRM, 예약 앱, 회원권 관리 프로그램, 웰니스 패스, AI 회의록 앱, 인바디 장비 판매/렌탈.

잡아야 할 포지션: "요가 수업이 끝난 뒤에도 남는 개인 수련 기록·공유 서비스."

| 비교 대상 | 한 줄 차이 |
|---|---|
| 바디코디 | 센터 운영을 기록 vs 몸소는 수업 경험을 기록 |
| 오붓/ClassPass | 수업에 가게 함 vs 몸소는 수업이 남게 함 |
| Tiro | 음성을 글로 바꿈 vs 몸소는 수업을 수련 기록으로 바꿈 |
| 인바디 | 몸의 스펙을 기록 vs 몸소는 수업의 맥락을 기록 |
| 포인티 | PT형 퍼스널 리포트 vs 몸소는 요가 수업의 언어·감각 기록에 특화 |

### 4. 두 관점이 만나는 지점

유동환은 "요가 현장에서 실제로 받아들일 수 있는가"(현장성·신뢰)를, 김성균은 "외부 심사와 파트너가 납득할 수 있는가"(포지셔닝·검증)를 묻는다. 몸소는 둘 다 통과해야 한다.

> 현장에서 신뢰 가능하고, 심사에서 설명 가능하며, 1차 MVP로 만들 수 있는 가장 작은 구조부터 간다.

### 5. 제품 원칙 — 데이터 계층과 공유 통제

몸소는 데이터를 하나의 파일처럼 다루면 안 된다. 원본 음성·전체 전사본은 기본 비공개, 회원에게 공유되는 것은 지도자가 검수한 개인별 리포트뿐이다. AI 자동 발송은 금지한다.

| 계층 | 기본 정책 |
|---|---|
| 원본 음성 | 기본 비공개, 단기 보관 후 삭제 또는 별도 동의 |
| 전체 전사본 | 내부 처리용, 회원 공유 금지 |
| AI 요약 후보 | 지도자 검수 전 자동 공유 금지 |
| 구조화 메모 | 지도자 검수 후 보관 |
| 개인 리포트 | 장기 보관, 회원 접근 가능 |
| 몸 데이터(인바디·웨어러블) | 동의 기반, 평가/진단 표현 금지 |

지도자는 기록 조각별로 공유 수준(개인 리포트 포함/제외, 내부 기록, 민감정보 삭제, 시퀀스 비공개, 마케팅 공개)을 선택할 수 있어야 한다.

### 6. MVP 정의

> 1:1 또는 소그룹 수업에서, 동의받은 음성을 기록하고, AI가 전사/요약 후보를 만들고, 지도자가 검수해 회원별 리포트를 공유하며, 이 기록을 장기 아카이브에 쌓는 서비스.

- 포함: 수업/참여자 등록, 녹음 동의, 녹음/업로드, 전사·요약 후보, 민감정보 후보 표시, 지도자 검수 화면, 개인 리포트, 회원 열람, 기록 누적.
- 제외: 예약/결제 CRM, 수강권·출입 관리, 대형 그룹 자동 리포트, 인바디 API 즉시 연동, SNS 전체 기능, 모든 장비 환경 지원.
- 첫 수업 유형 우선순위: ① 빅블루 내부 1:1 → ② 빅블루 2~4인 소그룹 → ③ 프리미엄 케어형 파트너 3~5곳 → ④ 대형 그룹은 후순위.
- 관리자용(원장·강사: 녹음·검수·공유 권한)과 소비자용(수련생: 리포트 열람, 녹음 권한 없음)을 논리적으로 분리.

### 7. 사용량·저장량 가정 (빅블루 실측 기반)

- 70~80분 수업 1회 ≈ 80MB. 빅블루 월 약 60타임 ≈ 약 5,000분 ≈ 약 5GB.

| 규모 | 월 녹음 시간 | 월 저장량 |
|---|---:|---:|
| 빅블루 1개 업장 | 약 5,000분 | 약 5GB |
| 10개 업장 | 약 50,000분 | 약 50GB |
| 20개 업장 | 약 100,000분 | 약 100GB |

API 비용은 저장량보다 처리 분수·실시간 여부·요약/재처리 횟수에 더 크게 좌우될 가능성이 높다.

### 8. Tiro 전략

Tiro는 공동창업자도 외주사도 확정 파트너도 아닌, AI 전사/요약 기술 파트너 후보이자 API 벤더 후보다. 미팅 결론은 '협력 확정'이 아니라, 몸소가 먼저 API 문서를 검토하고 PoC 기간·사용량·비용·요청 조건을 문서화하면 내부 논의가 가능하다는 것. Tiro 측은 공동개발 수준의 참여는 어렵다고 선을 그었다.

> Tiro와 협력하되, Tiro가 안 되면 멈추는 구조로 만들면 안 된다.

우선순위: REST/API·Webhook/export·Custom template(높음) > MCP(중간) > CLI(낮음~중간). 초기 협상에서 지분·독점권·매출 쉐어·공식 파트너 표현은 꺼내지 않고, 제한된 PoC 크레딧/할인, 성공 시 공동 사례화만 제안한다.

### 9. InBody 전략

인바디는 본체가 아니라 첫 정량 신체 데이터 파트너다. 초기에는 API가 아니라 결과지 업로드·이미지/PDF·수동 입력으로 PoC를 시작하고, 선정 후 CSV/export/API 연동을 협의한다.

- 좋은 표현: "인바디는 정량 참고값, 몸소는 수업 맥락 기록."
- 피해야 할 표현: "요가로 체성분이 개선된다", "몸소가 인바디 변화를 만든다", "인바디 API 즉시 연동", "Tiro와 공식 파트너십 체결", "국내 요가원 TAM 확정."

### 10. 시장과 BM

첫 시장은 국내 모든 요가원이 아니라 1:1·소그룹·프리미엄 케어형 스튜디오, 필라테스/PT/소매틱스/명상 센터 등 장기 기록 가치가 높은 업장이다.

| 상품 | 가격 방향 |
|---|---|
| PoC/파운딩 스튜디오 | 무료~월 4.9만 원 |
| Software Basic | 월 4.9만~9.9만 원 |
| Studio Pro | 월 15만~30만 원 |
| InBody Partner Package | 월 30만 원 이상 |
| B2C 기록권 | 월 5천~7천 원 |

매출 계산식:
```text
연매출 = 업장 수 × [B2B 월 구독료 + 업장당 B2C 유료 수련생 수 × 7,000원] × 12개월
```

B2B 월 30만 원 기준 업장당 연매출: 5명 402만 원 → 10명 444만 원 → 20명 528만 원 → 30명 612만 원.

3년 차 시나리오:

| 시나리오 | 업장 수 | 연매출 |
|---|---:|---:|
| 보수 | 300개 | 약 12억~16억 원 |
| 기준 | 500개 | 약 20억~26억 원 |
| 공격 | 1,000개 | 20명 기준 52.8억 원 |
| 공격 60억 조건 | 1,000개 | 30명 기준 61.2억 원 |

핵심: '연 60억 원'은 반드시 `1,000개 업장 + 업장당 유료 수련생 30명`이라는 조건을 붙여야 한다. B2C 월 7,000원은 저장비가 아니라 '개인 수련 기록권'으로, B2B 월 30만 원은 기록 인프라·리포트·현장 세팅·장비 옵션 결합으로 방어된다.

### 11. 리스크와 대응

- 개인정보/민감정보(통증·감정·타인 정보 혼입, 전사문이 음성보다 더 민감) → 수업별 명시 동의, 기본 비공개, 지도자 검수 필수, 민감정보 후보 자동 표시, 삭제 경로.
- 지도자 IP(시퀀스·큐잉·TTC 유출) → 시퀀스 비공개 옵션, 원본/전사본 다운로드 금지, 권한별 export 제한.
- Tiro 의존(비용 상승·협력 실패 시 제품 정지) → Tiro와 OpenAI/Whisper/온디바이스/수동 업로드 대안 병렬 검토.
- InBody 과대해석(체성분 개선 보장처럼 보임) → '정량 참고값'으로만, 진단/치료/개선/처방 표현 금지.
- 제품 범위 과대 → 첫 MVP는 `녹음 → 전사/요약 후보 → 지도자 검수 → 개인 리포트 → 누적 기록`만, 예약/결제/수강권·인바디 API는 후순위.

## 몸소에의 함의

- **카테고리명 고정**: 추천 '수업 맥락 기록 레이어'(보조: 웰니스 수업 기록 SaaS; 영어 보조: Voice Auto Archiving for Embodied Coaching).
- **하지 않을 것 명시**: CRM·결제·수강권·SNS·인바디 API 즉시 연동·대형 그룹 자동 리포트.
- **역할 분담**: 유동환은 요가 도메인·첫 PoC 수업·동의 문구·리포트 톤·FGI; 김성균은 Tiro API 검토·비용표·발표 논리·리서치 근거·기술 대안 비교.
- **다음 2~4주 액션**: 빅블루 1:1 수업을 룸마이크·스마트폰·핀마이크 중 최소 두 방식으로 비교 녹음하고 20/60/90분 샘플로 전사 품질·비용을 보며, 동의 문구·검수 리포트 템플릿·민감정보 제거 체크리스트·개인 리포트 1페이지 샘플을 만든다. Tiro에는 사용량 구간별 견적과 데이터 처리 조건을, InBody에는 결과지 업로드 기반 PoC를 제안한다.

최종 방향: 몸소가 만들어야 하는 것은 전사 기술이 아니라 '책임 있는 기록화'다.

```text
수업 전 동의 → 수업 음성 기록 → 전사/요약 후보 생성
→ 개인정보·타인정보·지도자 노하우 필터링 → 지도자 검수
→ 회원별 개인 리포트 → 장기 수련 아카이브 → 다음 수업·재방문으로 연결
```

지켜야 할 세 문장: ① 요가를 숫자로 환원하지 않는다 ② 수업 원본을 콘텐츠처럼 뿌리지 않는다 ③ 선생님과 수련생 사이의 사적 소통을 안전한 장기 기록으로 바꾼다.
