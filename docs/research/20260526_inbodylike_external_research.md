# InBodyLIKE 외부 리서치 — 몸소를 "인바디 데이터 × 수업 기록 계층"으로 재포지셔닝하기 위한 외부 근거 모음

> TL;DR: InBody는 정확도·재현성과 데이터 자산·글로벌 확장을, Bluepoint는 문제 정의·PoC·시장/투자 연계를 중시한다. 몸소는 "요가 앱"이 아니라 "인바디 정량 데이터 + 수업 언어 데이터를 결합한 웰니스 기록 계층 SaaS"로 말해야 하고, 빅블루 요가 현장 PoC가 사업계획서의 중심이 되어야 한다.

- 작성일: 2026-05-26
- 질문: InBody x Bluepoint의 오픈이노베이션 프로그램 InBodyLIKE가 무엇을 평가·중시하는가, 그에 맞춰 몸소를 어떻게 포지셔닝·서술할 것인가.

## 배경

InBodyLIKE는 InBody의 정밀 측정 기술과 신체 데이터 자산을, 스타트업의 민첩성과 결합해 헬스케어 혁신을 만드는 오픈이노베이션 프로그램이다. 핵심 키워드는 Strategic Fit, Trusted Growth, 신뢰 기반 파트너십, 인바디 인프라 활용 PoC, 제품·서비스·사업화 멘토링, 투자 및 TIPS 연계, 글로벌 확장 가능성이다.

- 모집 대상: 헬스케어/라이프스타일 기술·서비스·제품을 보유한 예비/초기 창업팀
- 모집 분야: ① 헬스 인텔리전스 기반 라이프스타일 혁신(체성분 데이터 활용 맞춤 건강관리·영양·운동, AI 분석 기반 건강 예측·신체기능 최적화) ② 데이터 기반 환자 관리 솔루션(질환관리 SaMD, 의료기관 연계, 임상 데이터 융합) ③ 글로벌 헬스케어 생태계·솔루션 확장 ④ 차세대 헬스케어 프론티어 기술(신규 생체신호 센싱, 웨어러블·반지형 등 신규 폼팩터)
- 일정: 서류 접수 2026-05-11~06-12, 평가·선발 2026-06-15~07-10, 최종 선발 2026-07-15, 육성 2026-07-20~11-13, 최종 성과 공유회 2026년 11월 4주차
- 제출물: 자유 양식 PPT 사업계획서를 PDF로 변환(필수), 3분 내외 피칭 영상 링크(선택). 포함 내용 = Problem, Solution, Scale-up, Team, 인바디와의 협업 포인트.

## 방법

InBody와 Bluepoint의 공개 자료(공식 사이트, 기술 설명, 연혁, InBody Report, 데이터 연동 문서, 포트폴리오 화면)와 공개 보도(전자신문·아시아경제·한국경제·뉴스1 등 InBodyLIKE 및 Bluepoint 오픈이노베이션 기사)를 수집·정리하고, 각 사항이 몸소 사업계획서에 주는 함의를 항목별로 해석했다. Bluepoint 포트폴리오 산업 분포는 공식 포트폴리오 화면의 공개 데이터를 2026-05-26 기준으로 직접 집계했다.

## 발견

### InBody가 중시하는 것

- **정확도·재현성을 기술 정체성으로 내세운다.** BIA의 기존 한계(단일 주파수, 경험식 보정, 전신 단일 원통 가정, 낮은 재현성)를 직접 해결했다고 강조한다. 부위별 직접 측정, 8점 터치식 전극, 다주파수 측정, 경험변수 배제, 반복 측정 재현성, 세포외/내수분·위상각 등 임상 지표 확장이 핵심 메시지다. → InBody는 "감"보다 "측정 가능한 근거"를 중시한다.
- **하드웨어 기반 데이터 자산으로 확장 중이다.** 전 세계 장비·클라우드로 체성분 데이터를 축적하고 InBody Report로 확장한다. 2024 InBody Report는 2018~2022년 17개국 데이터를 분석했고, 2023년 8월 기준 전 세계 체성분 데이터가 1억 건을 넘었다고 밝힌다. → 단순 장비 회사에서 데이터 기반 솔루션 회사로 이동 중.
- **데이터 연동·SW 생태계를 갖췄다.** Data Integration 문서에서 InBody Cloud Server, Web API, OAuth API, Open Protocol, LB120, LBS, SDK, 맞춤형 데이터 솔루션을 제시하며, LookinBody Web으로 InBody 데이터를 어디서든 관리·상담에 활용한다. → "연동 가능합니다" 수준으로는 약하고, 어떤 데이터가 들어와 어떤 화면·피드백으로 바뀌는지까지 설계해야 한다.
- **글로벌 네트워크·해외 확장을 강조한다.** 미국·일본(2000), 중국(2008), 말레이시아·인도(2016), 유럽(2017), 멕시코(2019), 영국(2020), 호주·미국 BWA·싱가포르(2023), 베트남 법인(2024) 등 거점 확장. → 몸소도 단일 요가원 앱이 아니라 필라테스·PT·명상·골프·테니스 등 코칭형 웰니스로 확장 가능한 구조로 제시해야 한다.

### Bluepoint가 중시하는 것

- **초기/딥테크 팀의 방향 설정.** 기술 신규성에만 매몰되지 않고 시장 문제를 풀도록 초기에 방향을 잡아주는 역할을 강조한다. 2026년 대표 인터뷰에서 Bluepoint를 "기술과 시장을 연결하는 시작 전문가"로 설명한다. → 기술보다 문제 정의와 시장 진입 순서가 먼저다.
- **PoC·시장 연결·투자 연계.** 전자신문 InBodyLIKE 기사에서 Bluepoint 수석심사역은 기술력만으로는 성장에 한계가 있고, 이 프로그램의 목적은 실제 사업 검증·시장 연결·투자 연계로 성장을 현실화하는 데 있다고 말했다. → 사업계획서에서 PoC가 가장 중요하다.
- **대기업·스타트업 역할 분담.** Bluepoint의 오픈이노베이션 사례는 대기업의 인프라·데이터·현업 네트워크와 스타트업의 민첩한 실험을 잇는 방식이다(한솔·GS·LG유플러스·포스코 등과 PoC·전용 펀드·사업화 검증 보도). → InBody에 "우리 앱 써주세요"가 아니라 "현장 PoC를 함께 설계하고 장비·데이터·운영 인프라를 연결해 달라"고 요청해야 한다.
- **포트폴리오 경향(2026-05-26 기준 322개 노출).** 헬스케어·AI·데이터·HW/SW 결합형 팀에 익숙하다. → "요가 앱"이라고만 하면 포트폴리오 언어와 멀어진다. "신체 데이터 + 수업 언어 데이터 + AI 피드백 + 현장 PoC + 확장 가능한 SaaS"로 말해야 한다.

## 몸소에의 함의

**포지션.** "요가 수업 후 AI 리포트를 보내는 앱"(약함)이 아니라 "InBody의 정량 신체 데이터와 요가 수업 중 발생하는 언어/피드백 데이터를 결합해, 요가·웰니스 현장에 부족했던 기록 계층을 만드는 데이터 기반 웰니스 SaaS"(강함)로 정의한다.

**Problem.** 요가·웰니스 수업은 체득·라포 중심이라 수업 중 중요한 피드백이 기록되지 않는다. 수련생은 자기 몸의 변화를 장기 조회하기 어렵고, 지도자는 다음 수업을 데이터 기반으로 잇기 어렵다. InBody 데이터는 몸의 스펙을 정량화하지만 수업 맥락과 연결되지 않으면 실제 경험 변화를 설명하기 어렵다.

**Solution.**
- 수업 전: InBody 측정 또는 기존 기록 확인 + 방문 전 AI 챗봇으로 수업 스타일·난이도·몸 상태 질문
- 수업 중: 지도자 설명·교정 피드백·수련생 반응을 음성/메모로 기록
- 수업 후: InBody 지표와 수업 언어를 묶어 개인별 AI 리포트 생성, 수련생은 다음 수업 전 "세컨드 바디" 기록 조회, 지도자는 지난 수업 핵심 포인트로 다음 수업 준비

**PoC.** 1차 실험장은 빅블루 요가. 검증 항목 = 수음 품질, 수련 방해 여부, 리포트 만족도, 재방문/재등록 영향, InBody 데이터 유입 시 상담 품질 상승 여부.

**InBody에 제안할 협업 포인트.** 비정형 웰니스 현장에서의 InBody 데이터 사용 PoC, 측정 데이터-수업 리포트 연결 UX 검증, 장비 설치/운영/렌탈 모델 검증, LookinBody/InBody Cloud 연동 검토, 헬스장·병원을 넘어 로컬 웰니스로 확장하는 레퍼런스 만들기.

**Bluepoint에 보여줄 것.** 초기 시장(요가원·웰니스 센터), 왜 지금(음성 전사·LLM 요약·클라우드 데이터 관리·InBody 연동), 왜 우리 팀(빅블루 요가·연희 요가 위크 운영 경험, 실제 수업 리포트 실험), 왜 확장 가능(개인 코칭형 업장 공통의 기록 문제), 무엇을 검증(현장 사용성·데이터 동의·지도자 업무 부담·수련생 리포트 가치).

**권장 사업계획서 목차.** 표지 → Problem(기록 계층 부재, 데이터-맥락 분리) → Solution(수업 전 챗봇 / 중 음성·피드백 기록 / 후 AI Body Report / InBody 연동) → Why InBody("InBody는 몸의 스펙, 몸소는 수업의 맥락") → PoC Plan(빅블루 요가 전/중/후 흐름, 측정 항목·성공 기준) → Market/Scale-up(요가→필라테스·PT·명상·골프·테니스) → Business Model(요가원 B2B 구독, 수련생 B2C 기록 구독, 장비/마이크/측정기 렌탈 옵션) → Team(도메인 운영자 + 개발/AI 파트너) → Request(InBody 데이터·장비·현업 멘토링·PoC / Bluepoint 액셀러레이팅·투자·TIPS·사업화 연결).
